# Obsidian + Claude Code 工作流

## 核心结论

**Obsidian 负责"记"，Claude Code 负责"想"和"做"。人类只做高价值筛选与决策，AI 接管所有繁琐节点的编织。**

---

## 三层架构

| 层级 | 组件 | 作用 |
|------|------|------|
| 底层 | Obsidian | 本地存储，Markdown，主权在你 |
| 中层 | Claude Code | AI 思考与执行引擎 |
| 顶层 | Skills | 自动化脚本，GitHub 下载 |

---

## 信息摄入

| 工具 | 用途 |
|------|------|
| Web Clipper | 网页/视频/公众号一键保存 |
| Daily Note | 自动日记模板 |
| Podwise | 播客转文字 |

**原则：一个入口进，后面自动分。**

---

## 杀手级命令

| 命令 | 功能 |
|------|------|
| `/todo` | 艾森豪威尔矩阵，今天优先级 AI 排 |
| `/weekly learning` | 每周复盘：认知变化 + 盲区 + 新想法 |
| `/summary` | 对话一键存档 |
| `/lint` | 知识库体检：矛盾检测 + 孤岛笔记 |

---

## Lint 检查项

- 概念定义有没有自相矛盾
- 有没有"孤岛笔记"（写了但没被引用）

AI 自动把碎片归类成：
- **Concepts** — 概念卡片
- **Summary** — 主题摘要

---

## 五个心得

1. **一个入口，多个终点** — Inbox 只管丢，后面自动分拣
2. **结构够用就行** — 先跑起来，边用边调
3. **AI 建议，你拍板** — 副驾驶不是代驾
4. **对话也是知识** — /summary 存档
5. **不绑定任何工具** — AGENTS.md 是通用协议
