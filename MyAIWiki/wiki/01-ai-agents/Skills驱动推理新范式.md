# Skills 驱动推理：TRS 用技能卡片降低推理成本

## 核心结论

TRS（Thinking with Reasoning Skills）的核心价值不是让模型「强行少想」，而是让模型「带着经验去想」。它把历史推理轨迹蒸馏成结构化 Skill Card，在新任务中检索注入，从而减少试错、回溯和重复验证，达到 Token 降本与准确率提升的双重收益。

**一句话：普通 CoT 是从零推理，TRS 是检索专家经验后再推理。**

## 解决的问题

推理模型通过长 CoT 提升数学、编程等复杂任务的准确率，但也带来三个工程问题：

- 输出 Token 暴涨，商业 API 成本上升
- 长思考增加端到端延迟
- 冗余试错、验证、回溯占据大量无效计算

传统压缩方案试图让模型「想短」，容易损伤难题准确率。TRS 的思路是给模型提供可复用的解题技能，让它少走弯路，而不是削弱推理深度。

## 方法框架

### 1. 离线蒸馏技能库

从历史问题的推理轨迹中提炼 Skill Card：

| 字段 | 作用 |
|------|------|
| Trigger | 判断技能适用场景 |
| Do | 给出最小可执行步骤 |
| Avoid | 避免常见反模式 |
| Check | 明确验证约束和不变量 |
| Risk | 标注边界情况和失败模式 |

成功轨迹提炼有效模式，失败轨迹提炼反模式与修正策略。

### 2. 在线检索注入

新问题到来时：

1. 根据问题检索相关技能卡片
2. 将技能卡片注入 Prompt
3. 让模型只采用直接适用的技能，忽略无关或冲突建议

数学任务偏向 BM25，代码任务更适合 BM25 + Dense Embedding 的 Hybrid 检索。

## 实验结论

### 数学任务

- Doubao Seed：准确率几乎不变，成本降低 53.8%
- GPT-4o-mini：准确率提升 1.8%，成本降低 6.9%
- GPT-OSS-120B：准确率不变，成本降低 16.9%

### 编程任务

- GPT-4o-mini：准确率 22.0% → 24.4%，成本下降 6.3%
- Doubao Seed-2.0：准确率 63.6% → 64.4%，成本下降 6.0%
- GPT-OSS-120B：准确率 54.2% → 58.3%，成本微升但收益明显

## 关键启发

### 1. 技能卡片不是普通 RAG

普通 RAG 提供相关资料，TRS 提供过程性指导：何时用、怎么做、避开什么、检查什么、哪里有风险。它更像「经验压缩包」，不是「上下文搬运工」。

### 2. 难题更需要技能，而不是更短的思考

TALE、CoD、NoWait 这类方法通过预算或措辞约束压缩思考，难题上容易崩。TRS 不压缩推理空间，而是提供导航地图，让模型更快找到有效路径。

### 3. 强模型蒸馏，弱模型执行

企业可以用强模型离线生成技能库，再让便宜模型在线调用，形成「大师经验，学徒执行」的成本结构。

### 4. Agent 记忆应从聊天记录升级为技能资产

对 Agent 落地来说，最有价值的记忆不是完整历史对话，而是可复用的任务经验：触发条件、操作步骤、避坑点、检查项、风险边界。

## 可迁移到研发工作流的方式

- 把 Claude Code 成功处理过的任务复盘成 Skill Card
- 将 Bug 排查、代码审查、重构、发布检查沉淀为技能库
- 针对 iOS / Android / C++ SDK 常见任务建立专用技能卡
- 在新任务开始时检索相关技能，减少重复试错
