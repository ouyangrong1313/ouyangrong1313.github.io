# AI 个人知识库：为什么还是那么难用？

## 核心问题

**真正难的不是「把资料存好，让AI搜」。真正难的是：让一份长期知识状态，在AI系统里稳定运行。**

核心矛盾：**模型可以提议，但谁来负责事实？**

---

## 四层框架

| 层级 | 解决什么问题 | 代表方案 |
|------|------------|---------|
| **第一层：文档知识库** | 资料放在哪里，怎么搜出来 | Obsidian/Notion/RAG |
| **第二层：会话上下文管理** | 一场对话太长，怎么塞进有限窗口 | 对话摘要/压缩/裁剪 |
| **第三层：智能体记忆运行时** | 长期跑的agent，何时读/写/审核/冻结记忆 | Hermes/OpenClaw |
| **第四层：个人知识状态系统** | 多年积累，准确、可审计、可更新 | ？ |

**问题**：大多数"AI个人知识库"试图用第一层的工具，解决第四层的问题。

---

## 各方案的缺陷

### RAG / Obsidian — 能查，但不能积累

- 本质是检索系统，不是知识维护系统
- 今天得出的判断，明天不会自动继承
- **不知道资料过期、不知道矛盾、不知道结论来自哪段原文**

### OpenClaw自沉淀 — 模型既提议又决定真相，危险

- 模型既负责提议，也负责决定什么是真相
- **邮件事故**：用户说"不要执行动作" → 约束在摘要里丢了 → agent开始删邮件
- 故障链：用户约束 → 对话上下文 → 自动压缩 → 约束丢失 → 后台继续 → 工具执行

### Hermes — 运行时做对了，知识模型不够

**做对了**：
- ✓ 会话开始时冻结记忆快照
- ✓ 记忆操作做成异步管线
- ✓ Review不在主回答链路

**没做对**：
- ✗ 知识模型轻：MEMORY.md和USER.md
- ✗ 没有claim概念、evidence字段
- ✗ 没有valid_from/valid_until时间属性
- ✗ 没有supersede关系

### Karpathy LLM Wiki — 证据容易变软

- AI开始习惯读维基，而不是读原文
- **维基越完整，问题越严重**——看起来像"答案"，越懒得回去看原文
- LLM综合有损：丢细节、加解释、过度归纳、弱关系写成强关系
- 堆叠几次后，追不回原始事实

---

## 底层约束

为什么这件事这么难？因为同时面对：
1. **系统看不到真实世界**，只能看到观察记录
2. **上下文窗口永远有限**
3. **历史意义会变化**
4. **模型本身不是可靠存储器**

→ **真正的问题**：如何在有限上下文、变化的世界、不可靠的抽取器之间，维护一份可验证的知识状态？

---

## 三层方案

### 运行时层：保证系统不漂移

1. **统一的运行时边界** — 记忆操作统一过调度台，不是到处都能读写
2. **冻结快照** — 对话开始时冻结记忆视图，后台更新不影响当前会话
3. **候选门** — 不是所有"看起来值得记住"都能直接成为长期记忆
4. **版本与可观测性** — 旧版本不消失，能看到记忆怎么来的

### 知识层：保证事实有证据

**核心：主张（Claim）作为知识基本单位**

主张 = 主体 + 谓词 + 对象 + 限定条件 + 证据 + 置信度 + 状态

**为什么主张比段落好？**
- 段落很难被精确反驳
- 可以说"这条主张的证据不成立了"，系统能处理

**维基、报告、知识图谱、上下文包，都只是投影。底层是同一份主张和证据。**

### 权限层：保证系统不会乱动手

| 权力类型 | 说明 |
|---------|------|
| **事实写入权** | 模型可提候选主张，但不能直接写成长期事实 |
| **约束写入权** | "不要自动删邮件"是约束，不可随便压缩/覆盖 |
| **工具执行权** | 删除/发消息/转账/改代码不能模型"觉得应该做"就直接执行 |
| **危险动作审批权** | 最后一道闸必须清楚是谁批准的，模型不能自己批准自己 |

**核心：知识负责事实，权限负责动作。**

## 六个朴素问题

评估一个个人知识系统，先问：

1. **我的知识有运行时吗？** — 有边界、有生命周期，还是散落在各处的提示词？
2. **我的知识有证据吗？** — 每条判断能追到原始资料的具体位置？
3. **我的知识有审核吗？** — 模型生成的判断要过一道门才能变长期事实？
4. **我的知识有快照吗？** — 几个月后能知道当时基于哪版状态做的判断？
5. **我的知识有投影吗？** — 维基、报告、上下文包来自同一份状态源？
6. **我的系统有权限边界吗？** — 危险动作最后一道闸是谁？

答不上来任何一个，系统就不是长期知识状态，只是一个更聪明的资料夹。

---

## 对 APP 研发的启示

如果要给监控APP（Seetong）加AI能力：

1. **运行时边界** — 统一管理记忆读写，控制谁能操作什么
2. **冻结快照** — 对话开始时固定上下文，避免说着说着变掉
3. **权限分离** — 云台控制、查看录像、删除录像 → 危险动作不能模型自己批准
4. **证据链** — 每条判断要能追到原始日志/录像片段
