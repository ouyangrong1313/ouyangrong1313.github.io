## 典型使用场景

### 场景一：文章写作助理

配置成文章写作助理，可以完成以下工作流：

1. **需求理解** → 读取预设的 Skill
2. **文件检查** → 检测文件名是否重名
3. **创建结构** → 按预定流程创建文件结构和大纲
4. **提交 PR** → 直接在 GitHub 发起 PR，并将 PR URL 发给用户

```json
// 实际工作流示例
{
  "step1": "读取写作Skill",
  "step2": "检查ID是否重名",
  "step3": "创建文件结构和文章大纲",
  "step4": "提交PR并返回URL"
}
```

**权限控制建议：**
- 可以配置成只提 PR，不直接操作分支
- 技术上做得到，但不推荐给太高权限

### 场景二：文章 Review

写完文章后让 Agent 帮助 Review：
- ✅ 检查错别字
- ✅ 指出表达不准确的地方
- ✅ **可以直接帮你改**（通过 PR 审核）

**价值：** 自己 Review 自己的内容往往发现不了问题，第三方视角更有帮助。

### 场景三：定时主动推送

利用 **Cron Jobs** 功能让 Agent 主动发消息：
- 每天早上推送 AI 领域重要信息
- 不只是被动等待指令，可以**自我激发**

**进阶用法：** 可以做成全天候随时能跟你主动联系的助理。

| 原则 | 说明 |

---

## Agent 自我进化

每个 Agent 都能自我进化：

1. **初期**：沟通要说比较细，需要明确指令
2. **后期**：很多事情不用多说，简单一句话就互相明白
3. **记录机制**：Agent 可以把你说的要求自己记录在 Skill 里，下次自动执行

**这就像和新同事合作的过程**——一开始互相不了解需要说得很细，熟悉之后简单一句话就能互相理解。

## 部署建议

### 硬件要求
- 旧 Mac 笔记本本地运行已能完成大部分需求
- 不需要高性能服务器

### 环境要求
- **物理隔离**：不要在日常使用的电脑上装 OpenClaw
- 对运维能力不够自信的话，轻易不要用云主机

### 模型选择
| 需求 | 推荐 |
|------|------|
| 性价比优先 | MiniMax（便宜） |
| 质量优先 | 更高级的模型 |

## 与人类协作类比

OpenClaw 越来越像**同事/合作伙伴**，而不只是工具：

- 大多数人类团队协作都是通过 IM 工具完成的
- OpenClaw 具备类似的工作方式
- 建立在安全前提下使用是关键
- **开源 + 免费** 更加难得
