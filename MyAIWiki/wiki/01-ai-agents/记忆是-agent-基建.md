# 记忆，是 Agent 基建

## 核心结论

**记忆正在从"产品差异化功能"升级为"Agent 时代基础设施"**

- 模型能力差距缩小后，记忆与 Context 成为核心竞争点
- 记忆所有权应属于用户，而非模型厂商
- OpenChronicle 提供本地优先、模型无关的开源实现

## 关键技术点

### AX Tree 优先方案

| 对比项 | AX Tree | 截图+OCR |
|--------|---------|----------|
| 成本 | 低（文本处理） | 高（图片处理） |
| 准确性 | 高（意图信号强） | 低（OCR 识别误差） |
| 结构化 | 是 | 否 |

- 混合方案：AX Tree 优先，截图兜底
- 触发机制：光标运动、页面滚动、应用切换时处理，避免无意义消耗

### Token 成本

- 轻度使用：≈$0.5/天
- 重度使用：$3-5/天
- 本地模型：仅需电费

### 记忆分类（7类）

扁平存为 Markdown，通过 MCP 协议暴露给 Agent：
1. 你这个人
2. 你的项目
3. 你常用的工具
4. 你关心的话题
5. 你联系的人
6. 你工作的组织
7. 你经历的事件

## 行业观点

> 记忆正在从「某个产品的差异化功能」逐渐变成「Agent 时代的基础设施」

- OpenAI Chronicle 推动行业，但商业限制使其困于自家应用
- 开源路径希望记忆属于用户，可接入任何 Agent
- 核心痛点：跨产品的记忆连贯性

## Proactive Agent

**定义**：能根据 context 主动判断、主动建议、主动行动的 Agent

**应用场景**：
- 代码中断后恢复
- 跨文档协作
- 设计稿修改延续
- 长期个人知识沉淀

> Agent 应该服务于人，去放大人的能力，不能让你变成各家厂商手里的资产
