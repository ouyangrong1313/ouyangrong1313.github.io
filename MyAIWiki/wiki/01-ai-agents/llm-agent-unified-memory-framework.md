# LLM Agent 统一记忆框架综述

## 核心观点

**Agent Memory 核心目标**：不要让模型"每次重读全部历史"，而是让系统主动维护一套记忆机制，在需要时取回相关信息，帮助 LLM 进行更可靠的长期推理。

---

## 问题背景

**Naive Long-Context Prompting 的问题**：
- 上下文窗口可能溢出
- Token 成本高
- 推理延迟增加
- 模型不一定能找到真正相关的证据

随着 GPT、Qwen、Claude 等大模型能力持续提升，LLM-based Agent 正在从单轮问答走向更复杂的**长期任务**：多轮对话、个人助手、游戏智能体等。

---

## 统一框架：四个核心组件

论文将 Agent Memory 拆解为四个核心组件：

| 组件 | 问题 |
|------|------|
| **Information Extraction** | 记什么？ |
| **Memory Management** | 怎么维护记忆？ |
| **Memory Storage** | 存在哪里、用什么结构存？ |
| **Information Retrieval** | 如何取回相关记忆？ |

---

### Information Extraction：记什么？

三种方式：

| 方式 | 说明 | 适用场景 |
|-----|------|---------|
| **直接归档** | 原始数据不做处理直接存入 | 数据量大、需要保留全量时 |
| **总结式提取** | 对信息进行压缩、总结后存入 | 空间有限、需要摘要时 |
| **基于图的提取** | 将信息组织成图结构 | 需要关系推理时 |

---

### Memory Management：怎么维护记忆？

五类操作：
1. **连接相关经验**：建立记忆之间的关联
2. **整合碎片记忆**：将零散信息聚合
3. **层级间迁移**：在不同记忆层级之间移动
4. **更新已有记忆**：修改或强化旧记忆
5. **过滤无用信息**：遗忘低价值内容

---

### Memory Storage：存在哪里、用什么结构存？

**两个维度**：

**组织结构**：
- **扁平式存储**（JSON、队列）
- **层级式存储**（长短期、树结构）

**表示方式**：
- **基于向量的存储**
- **基于图的存储**

---

### Information Retrieval：如何取回相关记忆？

四类检索方式：

| 检索方式 | 说明 | 优点 | 缺点 |
|---------|------|------|------|
| **词汇匹配检索** | BM25、Jaccard | 精确 | 语义理解差 |
| **向量检索** | 向量余弦相似度、ANN | 语义匹配强 | 需要额外存储 |
| **结构检索** | 图/树中邻居扩展、图遍历 | 关系清晰 | 实现复杂 |
| **LLM辅助检索** | LLM 参与判断相关性 | 智能 | 成本高 |

---

## 实验设计

**数据集**：
- **LOCOMO**：人类长期对话记忆数据集
- **LONGMEMEVAL**：用户与 AI 长期交互记忆数据集

**评测范围**：统一复现并比较 **10 个代表性 Agent Memory 方法**

**评测维度**：
- 整体性能对比
- Token 消耗及性能-成本权衡
- 上下文扩展性分析
- 证据位置敏感性分析
- 不同底层 LLM 下的表现对比

---

## 主要发现

### 发现1：层次化方法更优

MemTree、MemoryOS、MemOS 等层次化或树状方法表现突出。**多层结构能够同时保留高层摘要和底层证据**，更适合复杂长期任务。

### 发现2：粗粒度处理降本提效

将多轮对话作为一个整体进行处理可以显著降低 token 消耗，适当的粗粒度处理反而可能提升记忆效果。

### 发现3：上下文扩展性有限

当上下文规模扩展到 **200%** 时，几乎所有方法都会出现性能下降。相比之下，采用更明确层次管理的方法通常更稳定。

### 发现4：证据位置敏感

多数方法存在证据位置敏感性：当关键证据位于更早会话时，更容易被后续信息干扰而检索失败。

### 发现5：底层 LLM 仍决定上限

从 Qwen2.5-7B 扩展到 72B 后，多数方法都有明显提升。

---

## 新 SOTA 算法

组合最佳实践：
- **MemTree/MemOS** 的树状组织能力
- **MemoryOS** 的分层存储架构

→ 低 token 开销的新框架

---

## 对 APP 研发的启示

对于监控类 APP（Seetong）这类需要长期用户交互的应用：
- 用户长期交互 → 需要记忆系统
- **层级式存储**适合多会话场景
- Token 成本控制是关键优化点
