# 2026 年了，你的文件管理还停留在新建文件夹吗？

## 核心结论

文件管理的本质是**信息管理**。约翰·惠勒的 "It From Bit" 观点：万物皆比特，具体写了什么不重要，传递了什么信息才是最重要的。

**一句话：文件是拿来用的，不是存起来供起来展示的。**

## 三类信息

| 类型 | 说明 | 管理重点 |
|------|------|----------|
| 工具类信息 | 高复用图表、音视频、文字素材 | 便于快速找到 |
| 任务类信息 | 大大小小需要跟进的任務 | 根据轻重缓急分门别类 |
| 观点知识类信息 | 文娱影视、课程、文章等 | 收集→过滤→整理→内化→输出 |

## Inbox 统一入口

**减少 Inbox 入口**：确保信息来源唯一性，避免在多个平台重复收藏

**工具推荐**：
- **Eagle**：文件类信息 Inbox
- **Notion**：知识类信息 Inbox（Quick Note + Web Clipper）

**核心思路**：统一入口 → 定期筛查 → 分类处理

## 统一命名规范

**命名维度**（可根据权重调整顺序）：
```
领域_项目_版本_执行人_标签_备注_地点_时间
```

**示例**：
- `DUBG_猿神_V2(漏配补)` — 配音游戏猿神第二版
- `DIR_广播剧_SP_FB_V1_LQ` — 导演广播剧剧本反馈第一版

**注意事项**：
- 避免使用：! @ # ￥ % & < > ? , { } ' 「 ` 及空格
- NTFS 系统文件名最多 255 字符（1汉字=2字符）
- 尽量避免中文（兼容性考虑）

## 文件夹管理两大法

### 杜威十进制图书管理法

用编号建立体系：`01 生活`、`02 工作`

**原则**：
- 用 01-99 而不是 1-99（排序更清晰）
- 文件夹层级不超过 6 层
- 每个文件夹内尽量不超过 99 个项目

### PARA System

| 缩写 | 全称 | 说明 |
|------|------|------|
| P | Project | 有明确目标和 Deadline 的项目 |
| A | Area | 需长期关注维护的领域 |
| R | Resource | 支撑 P 和 A 的知识储备 |
| A | Archive | 已完成但可能需召回的内容 |

## 输出倒逼输入

**核心观点**：储存管理不是目的，在合适的时空场景准确快速找到合适的文件才是。

**费曼学习法**：教一个完全不了解这个知识点的人，能把他教明白，你自己自然也就懂了。

**最终目标**：复用复用再复用，直到内化成本能，张口即来。

## 321 备份原则

| 原则 | 说明 |
|------|------|
| 3 份副本 | 原始数据 + 2 份副本，三份同时丢失概率极低 |
| 2 种存储介质 | 内部硬盘 + 外部介质（U 盘/移动硬盘/ NAS） |
| 1 份异地存储 | 物理距离，防止自然灾害造成数据团灭 |
