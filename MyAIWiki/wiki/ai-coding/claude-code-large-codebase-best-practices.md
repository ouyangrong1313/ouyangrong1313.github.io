## 核心结论

Claude Code 在大型代码库中的表现，**Harness（基础设施）比模型本身更重要**。模型是大脑，Harness 是肌肉记忆——大脑再聪明，没有肌肉记忆也快不起来。

---

## 关键要点

### 1. 五大扩展点分层架构

| 组件 | 作用 | 加载时机 |
|------|------|---------|
| CLAUDE.md | 全局/局部规范 | 每次会话 |
| Hooks | 自动化脚本 | 事件触发 |
| Skills | 可复用专业知识包 | 按需加载 |
| Plugins | 技能+钩子+MCP 打包 | 配置后可用 |
| MCP Servers | 连接内部工具 | 配置后可用 |

### 2. Agentic Search 优于 RAG

Claude 不依赖向量索引，直接在最新代码库上工作。每个开发者的实例都是实时的，避免了 RAG 系统的陈旧检索问题。

### 3. 大型代码库导航六模式

1. CLAUDE.md 精简且分层（根目录放核心，子目录放局部）
2. 在子目录初始化，不在根目录
3. 测试/lint 命令限制在子目录级别
4. `.ignore` 屏蔽构建产物
5. 目录结构不清晰时制作"代码库地图"
6. LSP 实现符号级搜索

### 4. 每 3-6 个月更新配置

模型进化后，曾为老模型写的 CLAUDE.md 规则可能变成绊脚石。需要持续盘点清理。

### 5. 组织保障

- 指定 DRI（直接负责人）或 Agent Manager
- 提前打磨基础设施，第一天提供完整体验
- 大型组织需跨部门联合工作组

---

## 常见误区

- CLAUDE.md 塞入太多内容（应该用 Skills）
- 用提示词要求 Claude 做本该脚本自动完成的事
- 基础配置没搞就急着建 MCP 连接
- 在同一会话里既做探索又做编辑（应该用 Subagents）
