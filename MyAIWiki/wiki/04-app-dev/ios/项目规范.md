# Seetong-iOS 项目规范

## 项目概述

| 属性 | 值 |
|------|-----|
| 项目名 | Seetong-App-iOS |
| 语言 | Objective-C |
| 框架 | QMUIKit（腾讯） |
| 架构 | MVC / MVVM |
| 最低 iOS | 13.0 |
| 双平台 | Seetong_tps（国内）/ Seetong_tps_abroad（海外）|

## 环境搭建

```bash
cd Seetong5.0_ios
pod install
open Seetong_tps.xcworkspace
```

**注意**：项目包含设备专用库（如音视频解码），需要在**真机**上编译运行。

## 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | ST前缀 + 大驼峰 | `STLoginViewController` |
| ViewController | VC后缀 | `STDeviceListVC` |
| Model | Model后缀 | `STUserModel` |
| View | View后缀 | `STLoginBottomView` |
| Category方法 | `st_`前缀 | `UIColor+STExtensions` → `st_colorWithHex:` |
| 常量 | `k`前缀 + 小驼峰 | `kNotificationName` |
| 成员变量 | `_`前缀 | `_userId` |

**禁止**：禁止使用拼音、禁止在代码中出现中文。

## 代码组织 (.m 文件)

每个 `.m` 文件使用 pragma mark 分类：

```objc
#pragma mark - Life Cycle
#pragma mark - UI
#pragma mark - Data
#pragma mark - Observer
#pragma mark - Notification
#pragma mark - Delegate
#pragma mark - Action
#pragma mark - Public Methods
#pragma mark - Private Methods
#pragma mark - Setter / Getter
```

## UI 和数据分离

| 方法类型 | 命名 | 职责 |
|----------|------|------|
| UI初始化 | `setupUI` / `createUI` / `setupViews` | 创建视图、设置约束 |
| 数据加载 | `loadData` / `configData` | 网络请求、数据解析 |
| UI更新 | `updateUI` / `refreshUI` | 数据绑定到视图 |

## 全局导入 (STPrefixHeader.pch)

通过 `Import.h` 统一导入全局依赖：

- QMUIKit
- MJExtension（JSON序列化）
- Masonry（Auto Layout）
- Logan（日志）
- 业务基础类（BaseModel、STNetworkManager等）

## 关键依赖

| 库 | 用途 | 导入方式 |
|----|------|----------|
| QMUIKit | UI组件库 | `#import <QMUIKit/QMUIKit.h>` |
| YTKRequest | 网络层 | `STNetworkManager`（已封装）|
| MJExtension | JSON转Model | `#import <MJExtension/MJExtension.h>` |
| Masonry | Auto Layout | `#import <Masonry/Masonry.h>` |
| MJRefresh | 下拉刷新 | `STRefreshControl`（已封装）|
| Realm | 数据库 | `STDataBaseManager`（已封装）|

## Targets

| Target | 说明 | 特殊SDK |
|--------|------|---------|
| Seetong_tps | 国内版 | 闪验登录、友盟统计、支付宝 |

## Git 工作流

- `develop`：主开发分支
- 特性分支：`develop_version号`（如 `develop_v8.3.9`）
- 通过 GitLab Merge Request 合并到 `develop` 和 `test`
