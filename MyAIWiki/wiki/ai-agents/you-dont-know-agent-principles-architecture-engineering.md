# 你不知道的 Agent：原理、架构与工程实践

## 核心结论

Tw93 的 Agent 工程实践系统性总结，涵盖 12 个关键主题。核心判断：**Harness 比模型更关键**，调试 Agent 应优先检查工具定义，评测系统出问题先修评测再动 Agent。

---

## 分类提炼

### 工程优先于智能
- Agent Loop 核心不到 20 行代码，循环本身稳定，新能力通过工具扩展/提示调整/状态外化接入
- 模型负责推理，外部系统负责状态和边界

### 五种控制模式
| 模式 | 适用场景 |
|---|---|
| Prompt Chaining | 线性流程，先大纲再正文 |
| Routing | 简单问题轻量模型，复杂问题强模型 |
| Parallelization | 高风险决策、多视角 |
| Orchestrator-Workers | 中央 LLM 动态分解委派 |
| Evaluator-Optimizer | 质量难以精确定义的任务 |

### Harness 四件套
验收基线 + 执行边界 + 反馈信号 + 回退手段

### 上下文分层
常驻层 → 按需加载(Skills) → 运行时注入 → 记忆层 → 系统层

### 工具设计三阶段
1. API 封装（粒度过细）
2. ACI（面向 Agent 目标）
3. Advanced Tool Use（Search + Programmatic Calling + Examples）

### 四种记忆
工作记忆 / 程序性记忆 / 情景记忆 / 语义记忆

### 多 Agent 协作顺序
协议先于协作，隔离先于并行

---

## 要点列表

- **Harness > 模型**：验证测试质量对成功率的影响往往超过模型本身
- **上下文工程**：防 Context Rot，分层管理，Skills 按需加载
- **ACI 原则**：面向目标而非 API，边界清楚，参数防错，示例驱动
- **工具调用**：准确率 72%→90% 只需加 1-5 个示例
- **评测关键**：transcript + outcome 两类覆盖，Pass@k / Pass^k 不能混用
- **安全优先**：白名单 + 路径隔离 + 审计日志，第一版就到位
- **工程顺序**：单渠道先跑通 → 安全先于功能 → 记忆整合早做 → Skills 先于工具 → 第一个失败就建评测
