# 2026 LLM工程师路线图

## 核心结论

**做 LLM 应用，早就不是会写 Prompt 就够了。** 真正的生产级 LLM 系统需要从 Prompt、RAG、上下文工程、微调、Agent、部署、优化到安全评测形成一整套工程能力。

普通人进入 AI 行业的机会主要不在算法岗，而在**业务落地和 AI 应用工程**。

## 8层能力框架

Avi Chawla 的路线图（作者基本认可）：

1. **Prompt Engineering** — LLM 旅程入口，最便宜的杠杆
2. **RAG Systems** — 当模型训练数据没有答案时，需要知识增强
3. **Context Engineering** — 承上启下，决定模型看到什么
4. **Fine-tuning** — 优先级最低，绝大多数企业不需要
5. **Agents** — 从回答系统进入行动系统
6. **LLM Deployment** — 生产问题才出现
7. **LLM Optimization** — 第一张账单让你意识到重要性
8. **Safety, Evals & Observability** — 可观测性增加 20% 成本

## 普通人学习路径（6层）

| 层级 | 内容 | 关键点 |
|-----|------|-------|
| L1 | AI项目认知框架 | 看懂行业、项目类型、伪风口 |
| L2 | Prompt/Workflow | 把业务流程拆成模型可执行任务链 |
| L3 | RAG/Context Engineering | 上下文组织比召回更重要 |
| L4 | AI Coding | 必选项，不是加分项 |
| L5 | Agent | 让模型调用工具、拆解任务、持续执行 |
| L6 | Evals/Observability | 评估效果、发现问题、数据飞轮 |

## 关键认知

- **碎片化学习是进不去 AI 行业的真正原因**
- **AI 应用不是 demo 跑通就结束了** — 稳定性、延迟、成本、监控都是工程问题
- **很多新东西是老问题的新解法** — 看懂本质就不焦虑
- **数据工程是核心** — 微调最难的不是代码，是数据
