## 一句话总结

**Claude Code 是 Labs Team 的"意外产物"，但真正让其变强的是底层模型；Boris 已半年没写代码、卸了 IDE、同时跑 5-10 个 Claude 实例；他的工作变成"写 Loops"；招聘看 Generalist / Builder；给创始人的建议是"少招人，多给 tokens"；最后的护城河不是品味，是"价值观"。**

## 关键观点

1. **Claude Code 是意外产物** — 2024 年底 Labs Team 探索未来产品形态；最初只能完成 10-20% 工作

2. **编程 = AI Safety 天然实验场** — 反馈机制清晰（运行/不运行）+ 解空间收敛 + 海量训练数据

3. **Claude Code 变强的核心是模型变强** — Sonnet 4 → Opus 4 → Opus 4.5；产品功能是"增量改进"

4. **生产力数据** — 工程师产出代码量 3 倍（实际远超）；新工程师上手从数周 → 2 天

5. **"我的工作已经变成写 Loops"** — Boris 去年 11 月卸载 IDE；不再直接 Prompt Claude；运行 5-10 个 Claude 实例监督

6. **Generalist 黄金时代** — 分工瓦解，每个人都是 Builder；AI 降低跨领域迁移成本

7. **Member of Technical Staff (MTS)** — Meta 学来的去层级化；避免 deference bias；用想法本身竞争

8. **少招人，多给 tokens** — 每个项目故意少 50% 人头 + 大量 tokens；upfront 升 / ongoing 降；pre-compiling 类比

9. **品味被模型侵蚀** — Boris 自己的函数式编程执念被模型证明是错的；现在几百个 Claude 跑产品探索，20% 想法是好的

10. **价值观护城河** — 人类最后剩下的是"如何成为好的存在"，跟教孩子是同一件事
