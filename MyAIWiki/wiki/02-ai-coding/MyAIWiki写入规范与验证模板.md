# MyAIWiki写入规范与验证模板

## 核心结论（一句话）

所有正式 AI 知识沉淀默认写入 `MyAIWiki`，而不是 `.omx/wiki`；每次写入后至少校验路径、索引或日志、以及目标页面可读性。

## 分类提炼

- 场景：Codex / OMX / 知识库落盘 / 文章编译
- 标签：#主题/AI-Coding #主题/效率 #手法/工程实践 #场景/技术博客
- 类型：操作规范 / 验证模板 / 默认约定

## 要点列表

1. **正式知识库唯一入口**
   - 默认路径：`/Users/topsee/ouyangrong1313/MyAIWiki`
   - 用户说“知识库”“AI知识库”“MyAIWiki”“编译进去”时，默认都指向这里

2. **`.omx/wiki` 不是正式知识库**
   - `.omx/` 只作为运行时状态区
   - 除非用户明确要求，否则不把正式条目写入 `.omx/wiki`

3. **知识写入的最小文件集合**
   - 文章摄取：`raw/{slug}.md`、`raw/{slug}-digest.md`、`wiki/{分类}/{slug}.md`
   - 归纳页面：至少写 wiki 页面、索引、日志
   - 小型配置结论：允许只写 wiki 页面、索引、日志

4. **知识写入后的最小验证**
   - 校验目标路径正确
   - 校验索引或日志存在对应条目
   - 校验目标页面可读

5. **什么情况下必须补日志**
   - 新增 raw / digest / wiki 页面时
   - 新增配置清单、方法论页、操作规范页时
   - 更新索引结构时

## 验证模板

### 模板 A：文章编译到 MyAIWiki
- [ ] 原文存在于 `raw/`
- [ ] 拆解存在于 `raw/`
- [ ] 编译页存在于 `wiki/`
- [ ] 对应分类 `index.md` 已挂载
- [ ] `log.md` 已记录
- [ ] 编译页可正常读取

### 模板 B：只新增 wiki 结论页
- [ ] 目标页存在于 `wiki/`
- [ ] 分类索引已挂载
- [ ] `log.md` 已记录
- [ ] 页面首屏内容可读取

### 模板 C：修正错误写入路径
- [ ] 错误路径内容已删除或明确废弃
- [ ] 正确路径已补写
- [ ] 索引 / 日志都指向正确路径

## 推荐执行顺序

1. 先确认目标路径  
2. 再写目标文件  
3. 再补索引和日志  
4. 最后做最小验证
