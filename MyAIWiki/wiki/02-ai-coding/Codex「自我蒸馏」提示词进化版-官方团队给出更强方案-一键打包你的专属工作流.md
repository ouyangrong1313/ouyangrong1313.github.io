# Codex「自我蒸馏」提示词进化版！官方团队给出更强方案，一键打包你的专属工作流

## 核心结论（一句话）

这篇文章给出了一套更可执行的 Codex 自我蒸馏框架：基于最近工作记录筛选高频且可复用的任务，再用最小形式把它们沉淀成 skill、subagent 或 automation。

## 要点列表

1. 自我蒸馏的目标不是泛化总结，而是找出值得长期打包的重复性手动流程。
2. 证据来源有明确优先级：最近会话和任务摘要优先，其次是 Memories 与 rollout 摘要，再到 Chronicle，最后核对现有 skills、subagents、automations。
3. 一个候选流程只有在重复出现、输入稳定、步骤可复用、输出明确、收益显著时，才值得被封装。
4. 封装形式应尽量小：流程手册用 skill，边界清晰的委派工作用 custom subagent，周期性检查或提醒用 automation。
5. 在创建任何新资产前，应先确认现有内容是否已经覆盖，优先复用或扩展，避免重复创建。
6. 推荐的执行顺序是：先产出候选清单，再只创建高置信度、缺失且易验证的项。
7. 对证据不足、过于一次性、过于敏感或范围模糊的工作，应明确标记为 skip，而不是勉强打包。
