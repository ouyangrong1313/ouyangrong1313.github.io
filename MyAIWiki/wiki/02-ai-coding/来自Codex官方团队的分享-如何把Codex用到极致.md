# 来自 Codex 官方团队的分享：如何把 Codex 用到极致

## 核心结论（一句话）

把 Codex 用到极致的关键，不是让它多写代码，而是把持久线程、外部工具、自动化、目标验证和共享记忆拼成一套持续协作系统。

## 要点列表

1. **Durable threads 把 Codex 从单次问答变成长期协作空间**
   - 置顶线程适合承载发布、审稿、监控、幕僚长等持续性工作
   - 线程会保留决策、偏好和进度，减少每次重复喂上下文

2. **Steering 和 Queuing 让人重新获得长任务控制权**
   - Steering 用于任务执行中途纠偏
   - Queuing 用于提前安排当前步骤后的后续动作
   - 这两者一起解决了“AI 会做，但人插不进去”的问题

3. **Codex 的真实触达范围必须超出代码库**
   - 浏览器、Chrome 登录态、桌面操控、MCP 连接器和外部服务一起形成完整任务链
   - 许多关键工作在变成代码之前，起点只是聊天消息、邮件或网页界面

4. **Automation 负责持续推进，Goal 负责明确终点**
   - 定时自动化适合周期性任务
   - 线程自动化适合沿着同一个上下文持续推进
   - Goal 必须绑定测试、基准、复现场景或端到端流程等验证器

5. **侧边栏让“产物审查”回到同一个工作闭环**
   - 文件、网页、表格、PDF、幻灯片可以和对话并排审查
   - 这样评论、修订和再生成不再需要跨应用来回切换

6. **共享记忆要外置到知识库，而不是留在聊天记录里**
   - 代码库存代码，知识库存滚动上下文
   - `AGENTS.md` 应明确规定如何记录待办、决策、阻塞、负责人和有用链接

7. **对个人工作台和团队流程的直接启发**
   - 给高频任务建立固定线程和固定职责
   - 给长任务定义 stop condition 和 verifier
   - 把可复用上下文沉淀进 Obsidian / Git / 文本知识库
