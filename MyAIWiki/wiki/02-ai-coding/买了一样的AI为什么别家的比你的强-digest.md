## 一句话总结

**模型是商品，skill 才是资产**——AI 公司的竞争优势来自把"公司最厉害的人的判断"打包成可被 agent 复用的 skill；落地路径不是"建平台/选大模型"，而是找到具体重复工作，把顶尖人做的事写下来。

## 关键观点

1. **公司最好的人都有"沉默专长"**
   - 顶级销售电话前会找：上次对话记录 / 真正的决策人 / 没被说出口的反对意见 / 三周前某人许下的承诺
   - 这些判断力公司每天都在依赖，却很难被捕捉
   - AI 公司开始叫它 **skills**

2. **接数据 ≠ 好的工作**
   - 第一堵墙：数据不全 / 数据不净
   - **第二堵墙**：agent 不知道你的公司**怎么做**这件事
   - 帮 agent 理解判断逻辑，而不只是给它信息访问权限

3. **Skill ≠ Prompt**
   - Prompt：单时刻指令（"这时做 X"）
   - Skill：可重复工作方式（"这类任务这样做"）
   - Skill 可含：指令 / 示例 / 模板 / 检查清单 / 脚本 / 参考资料 / 经验法则

4. **技能库 vs SOP 库**
   - SOP 库：写下来 → 放进共享盘 → 没人读 → 6 个月后过期
   - 技能库：写下来 → agent 用 → 看结果 → 改 → 写下来 → ...（**关键区别是闭环**）

5. **模型是商品，skill 才是资产**
   - 模型越通用 → 差距越来自 skill
   - 真正的护城河：你喂给模型的特定知识

6. **怎么开始 — 从工作本身开始**
   - 找到重复出现的工作（销售 / 客户研究 / 支持 / PRD / 事故复盘 / 合同 / 预测 / 上线 / 竞争分析 / 发布说明）
   - 问团队里最好的人 5 个问题（首先注意什么 / 什么被忽略 / 哪些例子 / 反复出现的问题 / 怎么定义成功）
   - 把它变成 skill → 投入使用 → 持续改进

7. **真正瓶颈是"让人坐下来写"**
   - 写 SKILL.md 文件不难
   - 难的是把隐性知识显性化
   - AI 提高"写下来"的回报，不解决"写"的动机
   - **真正先行者 = 已有 onboarding 文档 / 流程手册 / 案例复盘习惯的公司**

## 核心论断

> **每家公司能用上的模型趋于相同，真正的分化只能来自你喂给它的特定知识。模型是商品，skill 才是资产。**

## 我的理解

- **"模型是商品，skill 才是资产"是 2026 年最锋利的 AI 战略判断**——和 [[从Prompt-Context到Harness-工程的三次进化与终局之战]] 同主线，但视角是"组织战略"而非"工程实现"
- **与 [[Agent Skills 系统性综述]] [[谷歌开源 agent-skills]] 互补**：那两篇是 skill 系统的"工程实现"，Hiten 这篇是 skill 系统的"战略必然性"——为什么所有公司**必须**做这件事
- **"沉默专长"对 Seetong 团队很关键**：张威 / 黄松佳 / 梁添 / 李深生 / 付林青 脑子里都有大量"看不到的判断力"——写下来就是 skill，不写就是流失
- **"先动手的那批人优势比想象的大"**——和 [[AI-Coding的顿悟时刻]] "工厂模式半年标配" 是同一信号：早期采纳者会拉开指数级差距
- **"让人坐下来写" 是真正的瓶颈**——跟 Seetong [[AGENTS.md]] "禁止重复造轮子 + 每次任务以 To-do List 开始" 是同一回事：纪律和沉淀习惯比任何工具都重要
