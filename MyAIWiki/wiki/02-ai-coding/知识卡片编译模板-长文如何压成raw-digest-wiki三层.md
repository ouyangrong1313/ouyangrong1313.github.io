# 知识卡片编译模板：长文如何压成 raw / digest / wiki 三层

## 核心结论（一句话）

长文进知识库时，不应该只存原文，而应该拆成 `raw` 保留上下文、`digest` 保留结构化提炼、`wiki` 保留可执行结论，这样 Codex 才能按需读取，而不是每次重啃全文。

## 分类提炼

- 场景：公众号长文、访谈、线程、深度教程、方法论文章
- 标签：#主题/AI-Coding #主题/效率 #手法/工程实践 #场景/技术博客
- 类型：卡片模板 / 知识编译流程 / 结构规范

## 要点列表

1. **`raw` 层保留原始上下文**
   - 存标题、作者、来源、日期、正文
   - 目标是“可回看”，不是“可直接复用”
   - 原则上不做抽象，只做必要清理和格式化

2. **`digest` 层负责结构化拆解**
   - 适合保留：
     - 3-5 个核心观点
     - 7 个角度 / 分析维度
     - 可传播钩子、可写作框架、对外表达句式
   - 目标是“便于复用”，尤其适合内容创作、培训、方法迁移

3. **`wiki` 层只保留可执行结论**
   - 一句话核心结论
   - 场景 / 标签 / 类型
   - 3-8 条关键要点
   - 相关链接
   - 目标是“给 Codex 和人类快速调用”，不重复长文细节

4. **不是每篇长文都值得完整三层**
   - 如果只是短笔记或单点结论，允许只写 wiki 页
   - 如果来源信息密度低，digest 可以更短，不必强行凑结构
   - 三层的目的是减负，不是制造文件

5. **判断是否适合卡片化的一个简单标准**
   - 内容长
   - 后续大概率反复引用
   - 含方法、框架、案例、判断标准中的至少两项

6. **卡片化后的最小验证**
   - `raw` / `digest` / `wiki` 文件存在
   - 分类索引已更新
   - `log.md` 已记录
   - wiki 页首屏可读，且比原文短得多

## 推荐模板

### 模板 A：方法论文章
- `raw`：原文全文
- `digest`：核心观点 + 7 角度 + 钩子
- `wiki`：结论、边界、可执行清单、相关链接

### 模板 B：访谈 / 线程
- `raw`：原始内容
- `digest`：观点聚类 + 反直觉句子 + 可传播表达
- `wiki`：提炼后的判断、原则、工作流启发

### 模板 C：工具 / 配置教程
- `raw`：原始教程
- `digest`：问题、方案、步骤、风险点
- `wiki`：适用场景、最小步骤、验证方式

## 什么时候不要做三层

- 只是临时问题，不需要沉淀
- 内容重复度很高，已有更好的同主题页
- 用户只要即时总结，不要归档
