# Codex任务交接与 `/new` 模板

## 核心结论（一句话）

一个任务结束后，先用一句交接摘要固化完成结果、修改范围、当前状态和下一步注意事项，再开启 `/new`，能明显减少旧上下文污染新任务。

## 分类提炼

- 场景：Codex CLI / 会话切换 / 任务收尾 / 上下文治理
- 标签：#主题/AI-Coding #主题/效率 #手法/工程实践 #场景/技术博客
- 类型：工作流模板 / 收尾模板 / 默认约定

## 什么时候用

- 一个功能或修复已经完成，准备开始下一个任务
- 当前会话已经积累了较多历史假设
- 接下来要切到新的需求、模块或问题

## 交接模板

```text
本次任务：<一句话总结>
修改文件：<文件1>、<文件2> ...
当前状态：<已完成 / 部分完成 / 已验证到哪一步>
下一步注意事项：<风险点、未验证项、人工确认点>
```

## 最小版模板

```text
一句话总结本次任务 + 修改文件 + 当前状态 + 下一步注意事项
```

## 使用顺序

1. 先完成当前任务的最小验证
2. 再写交接摘要
3. 然后开启 `/new`
4. 在新会话中贴入交接摘要，再继续下一个任务

## 这一步主要解决什么问题

1. 避免旧假设污染新任务
2. 降低新会话重新解释背景的成本
3. 让“完成”不只停留在聊天感觉，而是留下一条可复用摘要

## 避免事项

- 不要把交接摘要写成长篇复盘
- 不要省略当前状态和注意事项
- 不要在未完成最小验证前直接 `/new`
