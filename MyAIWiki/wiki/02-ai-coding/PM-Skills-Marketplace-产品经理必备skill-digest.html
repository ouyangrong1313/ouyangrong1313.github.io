核心观点 5 条

  1. “给文字” vs “给结构化执行路径”是 AI 工具的分水岭
  2. Skill = 独立方法论;Command = Skill 组合的流程
  3. AI 工具的成熟度看”领域方法论编码”
  4. AI 不替你做决定,只带你把方法论走一遍
  5. 方法论的护城河是基础认知

7 个分析角度

  1. 工具论 vs 方法论 — Skill 的本质是领域专家把”做事路径”代码化
  2. Claude Code 生态外溢 — Slash + Skill 成为 AI Coding 事实标准
  3. 产品经理最快被 AI 改造 — 80% 工作是结构化文本输出
  4. 标星 ≠ 采用率 — 16k 星是需求脉搏,不是使用证据
  5. Skill 生态在 jQuery 插件时代 — 协议有了,基建没有
  6. 方法论壁垒被抬高 — 会驾驭 Skill 的人价值跃迁
  7. 个人 Wiki 升级路径 — 从”可读”到”可调用”

21 个开头钩子(精选)

痛点 / 对比类

  • 我试过用好几个 AI 来写 PRD,直到我发现:它给的是文字,不是方法论
  • 通用 AI 给你模板;垂直 Skill 带你走完整流程。这是 2026 年 AI 工具的真正分水岭
  • 如果你的 AI 工具还在给你”内容”,那它就还停留在 2024 年

数据冲击类

  • 3 个月,16000 星,9 个插件 —— Slash command 正在从 Claude Code 蔓延到整个 AI 工具圈
  • /discover /lean-canvas /pre-mortem —— 这些斜杠命令正在重写产品经理的工作流

焦虑共鸣类

  • 做产品的同行注意了:你的工具链正在被 16000 个 GitHub 用户重写
  • 当 AI 能把 Lean Canvas、OST、Pre-mortem 都装进一个斜杠命令,产品经理的边界在哪里?
  • 会用 AI 的人在涨薪,不会的在被替换 —— 中间那批”只会填模板”的人,正在悄悄消失

反向揭秘类

  • 3 个月 16k 星,但真正用起来的可能不到 200 个 —— 这才是 GitHub 生态的真相
  • 装 9 个插件、复制 4 次目录、Windows 还会崩 —— 2026 年的 Skill 生态,就是 2008 年的 jQuery 插件
  • SKILL.md 协议有了,但分发、版本、依赖都没解决 —— Skill 的 npm 时代还没到

认知翻转类

  • Skill 把执行平民化了,但把”懂方法论”的门槛抬高了 —— 这是 AI 时代的新阶级分化
  • AI 向你提问,你答不上来 —— 这一刻你才知道,自己缺的不是工具,是基础认知
  • 你以为知识库是用来读的,其实它应该是用来”调用”的。这一字之差,决定了你能不能驾驭 AI
  • 如果你的 Wiki 只能”看”,那它就还是博客;能被 / 调用,它才是 Agent 的弹药库

自我经历类

  • 我的知识库有 200 篇文章,但我从来没”调用”过它 —— 直到我看到 PM Skills 的 Slash 化设计
  • 把开头钩子库变成 /hook-anxiety,把文章结构库变成 /structure-discover-solve —— 这才是知识库的下一代形态

趋势宣告类

  • 你以为 Slash command 是 Claude Code 的玩具,其实它是下一代 AI 工具的事实标准
  • 为什么 Anthropic、OpenAI 没人做 Skill Marketplace?因为协议简单,基建难

反直觉类

  • 为什么产品经理的工具链改造,比工程师还快?因为 80% 的产品工作是结构化文本输出
  • 开源标星是需求的脉搏,不是采用率的证据。怎么辨别一个 AI 项目是真火还是 PR?
