## 一句话总结

这篇文章真正有价值的，不是“Codex 比 Claude Code 更强”，而是它展示了一个更重要的趋势：**AI 编程工具正在从聊天框升级为 Agent 工作台，普通人该学的是项目式协作、上下文文件化、Skill 沉淀和多线程调度。**

## 关键观点

1. **起点不是 Prompt，而是项目目录**
   - 上下文应该沉淀为文件
   - 文件应该进入一个可持续工作的项目现场

2. **权限决定自动化程度，但必须有边界**
   - 实验项目可放开
   - 生产/客户项目必须有沙盒和边界

3. **Preview / Steer 缩短了反馈回路**
   - 可以边跑边纠偏，而不是等做完再改

4. **Plugin 连接能力，Skill 沉淀流程**
   - Prompt 是消耗品
   - Skill 是资产

5. **Computer Use 让 Agent 开始操作软件界面**
   - 价值不在于一步到位，而在于先替人接管大量机械动作

6. **多线程调度是新工作习惯**
   - 一条任务线一个上下文
   - 把高质量任务包交给不同 Agent 异步推进

7. **做产品要先打通主链路**
   - 先能用，再好看
   - 先跑通注册/登录/保存，再补边角和安全

## 我的理解

- 真正的 AI 协作能力，开始从“会不会写 Prompt”转向“会不会搭工作台”
- 人类越来越像 Agent 团队负责人：定目标、划边界、做反馈、验收资产
- Skill 和文件化上下文会成为普通人用好 Agent 的关键复利点
