## 一句话总结

作者公司全面切到 Codex（每人每月 200 美元），从工厂模式 / Scrum 反思 / 架构权重 / 裁员真相 4 个维度分享了 AI Coding 真实感触：**当执行成本越来越低，研发团队最大瓶颈只剩需求定义 + 架构设计；当稀缺资源从工程师时间变成正确判断，Scrum / 站会 / Story Point / PM 自然到该重构的时候；AI 裁员的真实财务模型是 4% 成本换 100% 产出，组织结构向两端收缩。**

## 关键观点

1. **工厂模式 = 半年内标配**：手工艺模式 → 工厂模式，AI/Agent 跑生产线，人退到后面设计生产线 + 设定规则 + 处理例外
2. **团队落地流程 = Spec → 反向采访 → LDD → tickets → PR**：
   - 先写 spec
   - Agent 反向采访人，把需求问清楚
   - Agent 生成 LDD（Lightweight Design Document），参考公司过去设计文档保持同种工程风格
   - 系统自动生成 tickets
   - 系统自动生成 PR
3. **Skill 本地化原则**："不是随便拿别人写好的模板来用，因为不同公司有自己的工程口味和产品手感"
4. **SCRUM 失效论**：Scrum 本质是管理不确定性，Agent 时代执行变得可预测，Sprint / Standup / Story Point / PM 依赖的前提正在被消解
5. **未来研发最大瓶颈 = 需求定义 + 架构设计**："决定一个系统未来几年命运的，从来都不是代码量，而是架构"
6. **方向错了执行越快越糟**："Agent 可以在极短时间内生成大量代码，但如果架构方向错了，那几十万行代码很多时候只是技术债务"
7. **AI 裁员的精确财务模型**：工程师年成本 50 万 + Agent 额度 1.7 万 = **4% 额外成本换 100% 产出提升**
8. **组织结构向两端收缩**："中间大量负责执行和传递信息的岗位会越来越少。留下来的，要么是负责方向和决策的人。要么是能够驾驭 Agent 的人"
9. **架构设计作者明确"暂时还不愿意完全交给 Agent"**
10. **裁员不是业务变差**："过去需要十个人完成的工作，现在五个人加一堆 Agent 就能完成"

## 我的理解

- **和 [[54万行代码的顿悟]] 互为补充**：
  - 那篇是"工程师个人范式"（Markdown 是新编程方式）
  - 这篇是"团队流程 + 组织重构"（Spec/LDD + Scrum 反思 + 裁员）
  - **合起来**：AI-native 团队需要个人范式 + 团队流程 + 组织结构三层都变
- **"Spec → LDD" 和 "Skillify it" 是两种互补的工程模式**：
  - Skillify：构建 → 技能化（结果导向，可复用）
  - Spec/LDD：意图 → 文档化设计（过程导向，过程可审）
  - **两者结合** = "用 LDD 把意图固定化 → 技能化执行"的完整闭环
- **"Skill 本地化"是最容易踩的坑**：很多团队直接套别人模板失败，是因为"口味"不对
- **"Scrum 失效"论不是否定 Scrum，是否定它依赖的前提**：前提变了（执行可预测），体系就要重构。不是"废除站会"，是"重新设计信息同步机制"
- **"4% 成本换 100% 产出"是说服老板的核武器**：不是道德判断，是经济判断
- **"组织向两端收缩" 是组织设计的真正方向**：留给我们的问题 —— 你属于哪一端？方向决策？还是驾驭 Agent？
- **架构能力 = 未来核心**：和"代码能力"是两条不同的职业线
- **对 Seetong 团队的启示**：5人+Agent 完成 10 人工作，可能是组织收缩的早期信号

## 关键术语

- **LDD**：Lightweight Design Document
- **Spec → LDD → tickets → PR 流水线**
- **Skill 本地化**：符合自己团队工程口味
- **组织向两端收缩**：决策者 + Agent 驾驭者
- **Scrum 失效论**：前提变了，体系要重构
- **架构权重 vs 代码量**：方向 > 速度
- **4% / 100%**：AI 成本与产出比的精确财务模型
