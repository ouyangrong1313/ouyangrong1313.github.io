## 核心要点

### 1. Claude Code 的诞生

- **2024年底**，Boris 加入 Anthropic Labs 孵化器，团队只有3人
- 最初产品就是 Claude Code、MCP 和 Claude Desktop App 三件套
- 团队一度被解散，2026年初由 **Mike Krieger**（Instagram 联合创始人、前CTO）牵头重组
- 做出来后**前6个月几乎没人用**，连 Boris 自己也只用它写10%的代码
- **爆发点**：2025年5月 Opus 4 发布后开始指数增长，每代新模型都让曲线再往上拐

> "我们其实是在做一个初期完全不具备 PMF 的东西。我们从始至终就是这个思路。"

### 2. "编程已经被解决" —— 对 Boris 自己100%成立

- 他的代码库（TypeScript + React）已经是模型写100%
- 单日合并PR记录：150个
- **但他承认有偏**：这是主流栈，不是C++老系统、不是SAP ABAP、不是游戏引擎

### 3. 手机里跑着几百个Agent的工作流

- 每天从手机完成大部分工作
- Claude App 常驻5-10个session，每个session里几百个Agent在跑
- 晚上还有几千个做深度任务
- 核心模式叫 **Loop**：用 cron 起定时循环，让 Claude 自动监控任务

常用 Loop 示例：
- 盯 PR → 自动修 CI、自动 rebase
- 盯 CI 健康 → 自动修 flaky 测试
- 每30分钟拉一遍 Twitter 上关于 Claude Code 的反馈，聚类整理

> "Loop 是未来。" —— Boris

### 4. 通才会越来越多

- 团队里的工程经理、产品经理、设计师、财务、用户研究员，**每个人都在写代码**
- 第二种通才更重要：跨学科的（既做产品又懂设计、又能做数据科学）

### 5. SaaS 的终结 —— 哪些护城河会被AI抹平

借用 Hamilton Helmer 的"七种护城河"框架：

**会被抹平的**：
- **切换成本**：模型可以帮你迁数据，300个工作流配好的系统一夜之间能迁完
- **流程效力**：Claude 4.7 已经能"hill-climb"（爬坡式优化）任何东西，你设定目标它自己迭代达成

**不变的**：
- 网络效应、规模经济、独占资源（专利、特许牌照）

### 6. 印刷术类比：最合适写会计软件的是会计师

> "我觉得最合适写会计软件的人已经不是工程师了，是一个真正懂业务的会计师。因为对领域熟得不能再熟，写代码反而是简单的部分。"

印刷术之后50年欧洲出版文献比之前1000年加起来还多。软件也会经历同样过程，但会快得多。

### 7. 真正的领先在组织流程，而非技术

- Anthropic 内部：所有 SQL、所有产品代码都由模型写
- 员工 Claude 之间通过 Slack 互相沟通，把不确定问题直接 ping 过去问
- 领先的关键是**组织怎么改造自己**，比追技术慢得多

### 8. 内部用的模型：Mythos + Opus 4.7

- **Mythos**：Anthropic 内部前沿模型，未公开。SWE-bench 93.9%、USAMO 97.6%
- 外界的 Claude Code 是用 Mythos 一起搭出来的

### 9. 产品 vs 模型

- 一年前 50/50，半年前也是 50/50
- 两年后？**"不知道，我们一次只做一周的计划"**
- 但产品永远重要：最终用户买的是"我每天用着舒服"

### 10. 云端 vs 本地 AI

> "这件事不重要。两年后模型会自己决定路由。"

### 11. MCP 与 Computer Use

- MCP 是答案：Claude.ai 的 Salesforce MCP，Cowork 也能用，所有入口都能用
- Computer Use 是 catch-all（兜底方案）：通过 Cowork 用能操作电脑任何软件，慢但4.7之后做得很好

### 12. 下一波"产品悬置"在 Claude Design

- 2026年4月17日发布，定位是"通过对话生成原型、幻灯片、营销页面的可视化工作台"
- 可以读取代码库自动套用 design system，一键导出到 Claude Code 或 Canva
- 直接定位成 Figma 和 Canva 的替代或补充

## 值得长期跟踪的三个预测

### 1. "编程已被解决"能否扩散到企业级代码库？

- Boris 的样本是 TypeScript+React，最受模型偏爱
- 真正考验这句话的是：SAP ABAP、嵌入式领域、底层系统、高合规场景
- 接下来一年能不能扩散，决定这句话是不是只对一小撮人成立

### 2. 切换成本和流程效力护城河是否真会塌？

- 这是 Anthropic 产品策略的底色，Cowork 就是赌这个
- 2026年2月软件股蒸发2850亿美元市值是市场初步反应
- 但企业IT采购周期通常24-36个月，真正兑现需观察接下来两年

### 3. 印刷术类比

- 数据有偏差（15世纪欧洲识字率约30%非10%，今日全球约90%非70%）
- 但方向对：软件构建会像识字一样普及
- 未展开的一面：印刷术也催生了最严格的审查制度、版权战争。AI写代码也不只释放创造力，也包括恶意软件、深伪诈骗的同步爆发

## 金句摘录

1. "产品悬置（Product Overhang）：模型已经能做一堆事情，但还没有任何产品把这些能力展现出来。"

2. "我们从始至终就是这个思路：为下一代模型做开发，不是为当前模型。"

3. "你设定一个目标，让它自己迭代优化，它最终就能达成那个效果。"

4. "如果用户得自己学会怎么调度这些工具，那其实是产品设计没做好，是我没做好。"

5. "未来最容易被取代的岗位，是那种'只会写代码、不懂任何垂直领域业务'的纯技术工程师。"

6. "模型的护城河不在技术，在组织流程。技术大家都能拿到，但要把整个公司从手写代码切到模型生成代码，这是组织行为的改造，比技术本身慢得多。"
