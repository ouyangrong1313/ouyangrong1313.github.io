Andrej Karpathy 那套知识管理系统看着眼馋，但不知道怎么落地？

这不是又一个"工具推荐清单"，而是一套真正跑通的工作流——Obsidian 当仓库，Claude Code 当管家，把你囤了多年的"死知识"盘活。

---

## 核心架构：三层大脑回路

### 底层：Obsidian
本地存储所有笔记，Markdown 通用格式。主权在你手里，不存在平台倒闭笔记全没的焦虑。

### 中层：Claude Code
AI 思考与执行引擎。读取笔记内容，帮你分析、提取重点、甚至直接干活。以前 Obsidian 是个静态笔记本，现在它成了会思考、会行动的助手。

### 顶层：Skills
从 GitHub 下载现成的自动化脚本，让 AI 自动跑重复任务。不需要会写代码，会用就行。

---

## 信息摄入

### Web Clipper
浏览器插件，一键保存网页、YouTube 视频（自动抓字幕）、公众号文章，直进 Obsidian Inbox。

### Daily Note
每天自动生成日记模板，睡眠、运动、灵感随手记，不用想格式。

### Podwise
把播客转成文字，无缝接入知识库。通勤听的东西，回家就能检索引用。

**核心原则：一个入口进，后面自动分。**

---

## AI 分析与处理

### /todo
输入后，AI 扫一遍你的日记和上下文，自动生成"艾森豪威尔矩阵"——重要紧急四象限，今天的优先级它帮你排，你只管拍板执行。

### /weekly learning
每周复盘。AI 翻完你一周的笔记，提炼三件事：
- 这周认知变了什么？
- 哪些盲区没注意到？
- 有没有新想法值得深挖？

### /summary
跟 AI 聊完天一键总结对话精华，自动存进知识库。灵感不浪费，对话也是资产。

---

## 整理与检查：给知识库做体检

### /lint
定期扫描。AI 检查：
- 概念定义有没有自相矛盾
- 有没有"孤岛笔记"——写了但没被任何笔记引用，等于白写

最后生成一份知识库健康报告，哪里堵了清清楚楚。

AI 还会自动把 Inbox 里的碎片信息，归类成：
- **Concepts**（概念卡片）
- **Summary**（主题摘要）

零散信息不是堆在那儿，而是织成一张能检索、能关联的网。

## Bonus：这套逻辑直接复用到一人 IP 创作

自媒体不是凭空想选题，而是用同一套系统跑通：

1. **全网选题扫描**，收集进 Input
2. **Claude 分析你的创作库**，从已有笔记里挖读者真正需要的内容——数据说话，不是拍脑袋
3. **AI 辅助 SOP**，初稿自动生成，你只负责精修和价值判断

**创作不是从零开始，是从你的知识里"长"出来的。**

---

## 跑通这套系统的五个心得

1. **一个入口，多个终点**。Inbox 只管丢，后面 AI 自动分拣，别在入口纠结分类。

2. **结构够用就行**。别在文件夹层级上追求完美，先跑起来，边用边调。

3. **AI 建议，你拍板**。别当甩手掌柜，关键决策必须过你的手，AI 是副驾驶不是代驾。

4. **对话也是知识**。聊完用 /summary 存档，今天的问题可能是明天的素材。

5. **不绑定任何工具**。AGENTS.md 是通用协议，换别的 AI 也能读，不怕被单一平台锁死。

## 一句话总结

Obsidian 负责"记"，Claude Code 负责"想"和"做"。人类只做高价值的筛选与决策，AI 接管所有繁琐节点的编织。
