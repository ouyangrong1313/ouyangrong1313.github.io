# Claude + Obsidian 第二大脑 - 拆解

## 核心观点

1. **第二大脑概念**：每天早上 Claude 已经知道你是谁、你在做什么项目、每个工具、每个任务、每篇文章、每个想法——这不是聊天机器人，是永不睡觉、永不忘记、每天使用都更聪明的第二大脑

2. **持久复合的 Wiki**：与 RAG 不同，不是每次问题从头发现知识，而是 LLM 增量构建和维护一个持久 wiki，交叉引用已在、矛盾已标记、综合已反映已读的一切

3. **三层架构**：Raw sources（不可变真相来源）→ Wiki（LLM 完全拥有）→ Schema（CLAUDE.md 等告诉 LLM 如何维护）

4. **三种操作**：Ingest（摄取新来源，10-15 wiki 页面）、Query（问 wiki，好的答案归档回 wiki）、Lint（每周健康检查）

5. **两个特殊文件**：index.md（面向内容，目录） + log.md（时间序，append-only 记录）

6. **维护即自动化**：人类放弃 wikis 因为维护负担增长快于价值。Claude 永久打破循环，维护只是一个命令

7. **Vannevar Bush 的 Memex 愿景**：私人、积极策划，文档间连接与文档本身一样有价值——他无法解决的是谁做维护，现在 LLM 解决了

## 分析角度

### 1. 产品定位角度
- 目标用户：知识工作者、开发者、研究者
- 核心价值主张：5 分钟建立第二大脑，终身受益
- 与 Notion/Roam 的差异化：AI 原生维护，Claude 自动做所有簿记工作
- 情感诉求："should be illegal" 标题制造好奇心和 FOMO

### 2. 技术架构角度
- 三层设计清晰解耦：Sources（只读）→ Wiki（AI 写）→ Schema（规则）
- Index + Log 双文件导航机制
- 可选工具链：qmd 搜索、Marp 幻灯片、Dataview 动态查询
- Git 版本控制免费获得

### 3. 用户体验角度
- 门槛极低：下载 Obsidian + Claude Code + 粘贴提示词 = 5 分钟
- 渐进价值：vault 不需要完整，只需要真实
- 无痛维护：维护只是一个命令，不再是负担

### 4. 内容营销角度
- 标题公式："X should be illegal" = 好奇心触发
- 数据背书：718万+ 查看、45932 书签、9595 喜欢
- 步骤清晰：截图 + 链接 + 代码块 = 易于实操
- 社区激活：198 回复、高互动

### 5. 商业模式角度
- 引流到 Obsidian（免费）+ Claude Code（免费）= 无需付费
- 潜在变现：知识付费（课程）、工具推荐、SaaS 封装
- 内容即获客：爆款推文带来大量追随者

### 6. 竞争格局角度
- Notion AI：文档管理，但缺个性化知识积累
- Roam Research：双向链接，但缺 AI 自动维护
- Obsidian + Claude Code：当前最强组合
- 核心壁垒：Claude Code 的编程能力使维护成本接近零

### 7. 未来演进角度
- 个人知识库 → 团队知识库（Slack 整合）
- 本地 → 云端同步（多设备）
- 单人 → 协作（Git 机制）
- LLM 主动推理（不只是被动响应）
