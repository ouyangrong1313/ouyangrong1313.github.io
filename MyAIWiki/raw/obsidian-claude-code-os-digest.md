# Obsidian + Claude Code 工作流拆解

## 核心观点

**Obsidian 负责"记"，Claude Code 负责"想"和"做"。人类只做高价值筛选与决策，AI 接管所有繁琐节点的编织。**

不是工具推荐，是一套真正跑通的个人知识管理 + 自动化操作系统。

---

## 角度一：为什么 Karpathy 方案难落地

### 核心洞察

Karpathy 那套知识管理系统看着眼馋，但不知道怎么落地？

问题不是缺工具，是缺一套**真正跑通的工作流**。

这篇文章给的答案：Obsidian（仓库）+ Claude Code（管家）+ Skills（自动化）

### 开头钩子

1. "Karpathy 的知识管理系统看着眼馋，但一动手就卡——你缺的不是工具，是工作流。"

2. "Obsidian + Claude Code 不是两个工具拼接，是一个操作系统：你记，它想，它做。"

3. "为什么你囤了那么多笔记却用不起来？因为只有'记'，没有'想'和'做'。"

---

## 角度二：三层大脑回路

### 核心洞察

三层架构：
- **底层：Obsidian** — 本地存储，Markdown，主权在你
- **中层：Claude Code** — AI 思考与执行引擎
- **顶层：Skills** — 自动化脚本，GitHub 下载，会用就行

以前 Obsidian 是静态笔记本，现在变成会思考、会行动的助手。

### 开头钩子

1. "Obsidian 是仓库，Claude Code 是管家，Skills 是自动化脚本——三层架构缺一不可。"

2. "你的笔记为什么还是'死'的？因为缺了中间那层——AI 思考与执行引擎。"

3. "不需要会写代码，Skills 让你直接用 GitHub 上的现成脚本——会复制粘贴就行。"

---

## 角度三：信息摄入的秘密

### 核心洞察

**一个入口进，后面自动分。**

- Web Clipper：网页/视频/公众号一键保存
- Daily Note：自动日记模板
- Podwise：播客转文字

别在入口纠结分类，后面让 AI 自动分拣。

### 开头钩子

1. "你还在为'这篇文章该放哪个文件夹'纠结？Obsidian 的 Inbox 就是用来丢的，后面让 AI 分类。"

2. "信息摄入的原则是：一个入口，多个终点。别在入口做分类决策。"

3. "播客听完就忘？Podwise 把播客转成文字，无缝进知识库——通勤听的东西回家就能检索。"

---

## 角度四：三个杀手级命令

### 核心洞察

- **/todo** — 艾森豪威尔矩阵自动生成，今天优先级 AI 排
- **/weekly learning** — 每周复盘：认知变化 + 盲区 + 新想法
- **/summary** — 对话一键存档，今天的问题可能是明天的素材

### 开头钩子

1. "输入 /todo，AI 扫完你的日记和上下文，输出艾森豪威尔矩阵——你只管拍板执行。"

2. "/weekly learning 不是周报，是认知升级的刻意练习——AI 帮你从一周笔记里挖出三件事。"

3. "聊完天不用手动整理，输入 /summary 一键存档——对话也是资产，不是聊完就忘。"

---

## 角度五：/lint 给知识库做体检

### 核心洞察

定期扫描两件事：
- 概念定义有没有自相矛盾
- 有没有"孤岛笔记"——写了但没被引用，等于白写

AI 还会自动把碎片归类成 Concepts 和 Summary。

零散信息 → 能检索、能关联的知识网。

### 开头钩子

1. "你有多少笔记写了之后从来没被引用过？/lint 告诉你哪些是'孤岛笔记'——写了等于白写。"

2. "知识库不是囤积，是织网。/lint 定期体检，让你的笔记从堆砌变成网络。"

3. "为什么你的笔记用不起来？因为缺一个'图书馆管理员'——/lint 就是 AI 扮演这个角色。"

---

## 角度六：复用这套逻辑做一人 IP

### 核心洞察

自媒体不是从零开始想选题，是从已有知识里"长"出来：

1. 全网选题扫描 → Input
2. Claude 分析创作库 → 挖读者真正需要的内容
3. AI 辅助 SOP → 初稿自动生成，你只负责精修

**数据说话，不是拍脑袋。**

### 开头钩子

1. "做内容不是凭空想选题，是用同一套系统从你的知识库里挖——数据说话，不是拍脑袋。"

2. "创作不是从零开始，是从你的知识里'长'出来的。这句话值一篇 10w+。"

3. "一人 IP 的护城河不是创意，是你的知识库厚度——Obsidian + Claude Code 就是你的创意工厂。"

---

## 角度七：五个心得

### 核心洞察

1. **一个入口，多个终点** — Inbox 只管丢，后面自动分
2. **结构够用就行** — 先跑起来，边用边调
3. **AI 建议，你拍板** — 副驾驶不是代驾
4. **对话也是知识** — /summary 存档
5. **不绑定任何工具** — AGENTS.md 是通用协议

### 开头钩子

1. "跑了这套系统一年，五个心得：入口简单、 结构够用、 AI 建议你拍板、 对话存档、 不绑定工具。"

2. "很多人问文件夹怎么分类，我的答案是：别在入口纠结，后面让 AI 分。"

3. "Obsidian + Claude Code 的最大价值不是知识管理，是让你从'知识的搬运工'变成'知识的主人'。"
