# Skills驱动推理新范式 — 拆解

## 核心观点

1. **推理模型正在遭遇 Token 通胀** — CoT 提升准确率，但长思考导致成本和延迟上升。
2. **TRS 的关键不是压缩思考，而是复用经验** — 将历史推理轨迹蒸馏成 Skill Card，推理时检索注入。
3. **结构化技能卡片比普通 RAG 更有效** — Trigger、Do、Avoid、Check、Risk 提供可执行的过程性指导。
4. **难题上收益更大** — 强制「想短」会让模型崩，技能卡片则像导航地图，减少试错路径。
5. **工程落地价值明确** — 强模型离线蒸馏技能库，弱模型在线执行，实现「大师经验，学徒执行」。

## 7 个分析角度

### 角度1：成本危机 — 推理模型的 Token 通胀
推理模型越强，思考越长；输出 Token 昂贵，使复杂任务成本飙升。

### 角度2：范式转移 — 从 CoT 到 Skill Recall
不是让模型每次重新推导，而是像专家一样调用沉淀过的解题技能。

### 角度3：知识结构 — Skill Card 的五字段设计
Trigger、Do、Avoid、Check、Risk 把经验压缩成可检索、可执行、可校验的格式。

### 角度4：工程架构 — 离线蒸馏 + 在线检索注入
强模型负责总结经验，线上模型负责调用经验，形成训练外的能力增强层。

### 角度5：效果验证 — 更少 Token 但准确率不降反升
数学和编程任务中，Token 消耗降低 6%-59%，部分模型准确率反而提升。

### 角度6：与 RAG 的区别 — 相关信息不等于过程指导
普通 RAG 提供上下文，TRS 提供步骤、陷阱、校验点和风险边界。

### 角度7：Agent 落地启发 — 技能库是 Agent 记忆系统的工程形态
TRS 可迁移到 Agent 工作流：把成功任务轨迹沉淀为技能卡片，让后续任务少走弯路。

## 21 个开头钩子

### 角度1：成本危机
1. "推理模型越来越聪明，但账单也越来越吓人。"
2. "为什么模型答对一道题，要先写一万字内心独白？"
3. "CoT 解决了准确率，却制造了新的 Token 通胀。"

### 角度2：范式转移
4. "真正的专家，不会每次都从零开始推导。"
5. "AI 推理的下一步，可能不是想得更久，而是记得更准。"
6. "让模型少走弯路，关键不是堵住思考，而是给它经验。"

### 角度3：知识结构
7. "一张好的 Skill Card，比一大段上下文更值钱。"
8. "Trigger、Do、Avoid、Check、Risk：这是 AI 经验的五件套。"
9. "经验不能只收藏，必须压缩成模型能执行的步骤。"

### 角度4：工程架构
10. "强模型当老师，弱模型当学生，这是企业降本的新路径。"
11. "离线烧钱总结经验，在线省钱复用经验。"
12. "TRS 给了一个黑盒模型也能增强推理的工程方案。"

### 角度5：效果验证
13. "Token 少了 59%，准确率却没掉，这听起来反直觉。"
14. "最有意思的是：TRS 不是牺牲准确率换成本，而是两头都要。"
15. "难题越难，技能卡片越像导航地图。"

### 角度6：RAG 区别
16. "普通 RAG 告诉模型‘资料在哪’，TRS 告诉模型‘该怎么做’。"
17. "相关上下文不等于可执行经验。"
18. "RAG 给知识，Skill Card 给动作。"

### 角度7：Agent 落地
19. "Agent 真正的记忆，不该只是聊天记录，而应该是技能卡片。"
20. "每一次任务轨迹，都可能变成下一次任务的捷径。"
21. "会复盘的 Agent，才有可能越用越便宜、越用越准。"

## 可借鉴到 MyAIWiki / Agent 落地的点

1. **把高价值工作流沉淀为 Skill Card**
   - Trigger：什么场景触发
   - Do：标准操作步骤
   - Avoid：常见错误
   - Check：验收标准
   - Risk：边界条件

2. **把 Claude Code 会话复盘转成技能库**
   - 不只保存原文和总结，还保存「这类任务下次怎么做」。
   - 适合沉淀到 `wiki/02-ai-coding/workflows/` 或未来的 `skills/` 专题。

3. **对 APP 研发 Agent 的启发**
   - iOS/Android/C++ SDK 的常见问题可以沉淀成技能卡片。
   - 例如：崩溃排查技能、Objective-C UI 改造技能、SDK 接口兼容性检查技能。
