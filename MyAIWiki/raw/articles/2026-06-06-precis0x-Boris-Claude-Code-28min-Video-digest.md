# Boris 28 分钟视频拆解 - 提示词真相

> 对应原文：[[articles/2026-06-06-precis0x-Boris-Claude-Code-28min-Video]]
> 推文一句话：**"Claude Code 作者 Boris Cherny 的 28 分钟免费视频，比 300 美元课程讲得更透"**

---

## 核心观点（4 个）

1. **真正的提示词专家是工具的创造者** —— Anthropic 工程师 Boris Cherny（Claude Code 主要作者）亲自下场讲提示词，相当于"造火箭的人告诉你怎么开"
2. **28 分钟就够讲透** —— 反常识：300 美元的课程前 10 分钟都不如他讲的一半，证明这个领域被过度商业化、注水严重
3. **核心四件套** —— 视频聚焦四件事：CLAUDE.md 文件、内存快捷方式、并行会话、提示模式
4. **免费即稀缺** —— 知识在 AI 时代被严重高估，作者愿意把核心方法论免费放出，是另一种维度的"信号"

---

## 5 个分析角度 + 钩子

### 角度 1：权威性反转（造火箭的人教你开火箭）

> 当工具的创造者亲自下场写教程，整个培训市场都被重新定价。

- 钩子 a：「Claude Code 工程师亲自教你写 prompt，这是 2026 年最被低估的 28 分钟」
- 钩子 b：「为什么 Anthropic 的工程师敢把核心方法论免费放出来？」
- 钩子 c：「他做的工具，他最懂。但几乎没人听他讲」

### 角度 2：免费 vs 付费（教育泡沫的刺破）

> 300 美元的课程讲不到的东西，一个 28 分钟的免费视频讲透了。知识付费这门生意，正在被 AI 原生时代的"作者亲自下场"肢解。

- 钩子 a：「我付了 300 美元学 prompt，结果发现真正的高手在 X 上发 28 分钟免费视频」
- 钩子 b：「知识付费的护城河，正在被作者本人击穿」
- 钩子 c：「不是课没用，是讲的人不对」

### 角度 3：四件套组合拳（CLAUDE.md + 内存 + 并行 + 模式）

> 一个视频里浓缩了 4 个核心方法：CLAUDE.md 沉淀、内存快捷方式、并行会话、提示模式 —— 每一个都解决一个具体的"卡点"。

- 钩子 a：「写 prompt 卡了 3 个月？这 4 件事你可能一件都没做对」
- 钩子 b：「为什么你的 Claude 总不听话？因为你少了一个 CLAUDE.md」
- 钩子 c：「别再让 Claude 重复犯错 —— 内存快捷方式的正确打开方式」

### 角度 4：提示词不是玄学，是工程

> 把"提示词"从"灵感型创作"重新定位成"工程化配置"，是 2026 年 AI 使用方式的最大分水岭。

- 钩子 a：「2026 年最被低估的技能：把 prompt 写成工程文件」
- 钩子 b：「为什么你的 prompt 经常失效？因为你没把它当配置」
- 钩子 c：「告别 prompt 玄学：从 CLAUDE.md 开始，4 步把它工程化」

### 角度 5：作者亲自下场 = 行业信号

> 当顶尖公司的核心工程师愿意 28 分钟免费讲方法论，传递的信号是：模型层护城河在变薄，"使用能力"才是下一个差异化点。

- 钩子 a：「Anthropic 工程师 28 分钟视频火了 88 万次，传递的信号比内容更值得读」
- 钩子 b：「当模型不再稀缺，使用模型的能力就成了新护城河」
- 钩子 c：「Boris 这条视频，可能比 Claude Code 4.0 发布更重要」
