## 问题背景

你可能遇到过这些问题：

1. **信息碎片化**：推特看到好文，一顿复制粘贴。想找出来学习，得手动翻半天。
2. **笔记孤岛化**：Notion 里存了几百篇笔记，但笔记之间是孤立的。不知道某个概念在其他地方出现过。
3. **AI无积累**：每次问 AI 问题，它都从零开始搜索，临时拼凑答案。没有积累，没有记忆，Token 还废掉一半。
4. **数据不安全**：笔记存在别人服务器上，哪天服务关了，数据就没了。

## 解决方案：四个核心优点

- **完全自动化**：你不需要手动整理笔记，AI 自动帮你做
- **本地存储**：数据永远属于你自己，不会上传到任何服务器
- **持久化积累**：知识会不断积累，不是每次从零开始
- **只需提问**：你只需要提问和探索，其他交给 AI

**核心链路**：文档导入 → AI 整理 → Wiki 生成 → 双向链接

## 三个工具，各有分工

### 1. Obsidian：笔记展示层

Obsidian 是一款本地双向链接笔记工具，完全免费，跨 Windows、Mac、Linux 三平台。

**核心特点：双向链接**

在笔记里输入两个方括号，比如 `[[Claude-Code-笔记]]`，Obsidian 会自动把它变成一个紫色链接。如果该笔记存在，点击就能跳转；如果不存在，点进去就会创建它。

**Graph View**：把所有笔记和它们之间的链接关系可视化成一张图，一眼就能看到知识结构，看到哪些节点是孤岛，哪些是枢纽。

### 2. Hermes Agent：自动化执行引擎

Hermes Agent 是 Nous Research 开发的自主 AI 代理，最大特点是内置学习循环，能从经验中创建技能、改进技能。

**承担功能**：自动化执行引擎

它内置了 `llm-wiki` skill，可以直接按 LLM Wiki 的文件结构规范操作知识库。

当你明确要求时，Hermes 会自动完成：
1. 提取文档中的关键实体（人物、工具、项目）
2. 提取核心概念（方法论、技术原理）
3. 创建结构化的 Markdown 文件
4. 添加双向链接连接相关概念
5. 更新知识库索引

**使用规则**：
- 说「写入知识库」→ 执行导入操作
- 说「结合知识库」→ 检索知识库
- 日常对话 → 不主动访问知识库

### 3. LLM Wiki：知识库标准

LLM Wiki 不是独立应用，而是一套知识库文件结构规范：

```
knowledge_base/
├── raw/sources/              # 原始素材
├── wiki/entities/            # 实体（人物、工具、项目）
├── wiki/concepts/            # 概念（方法论、原理）
├── wiki/index.md             # 知识库索引
└── wiki/log.md               # 更新日志
```

**核心思路**：让 AI 增量构建持久化 Wiki

导入文档时，系统会真正理解文档，提取关键实体、概念、关系，生成或更新对应的 Wiki 页面。这些页面保存在本地，随时间推移越来越丰富，页面之间形成引用和关联。

## 完整工作流

### 第一步：下达指令
比如你说「把这篇 AI 写小说的文章写入知识库」。

### 第二步：Hermes 自动整理
Hermes 通过内置的 `llm-wiki` skill 自动完成：
- 读取文档内容
- 提取关键实体和概念
- 创建结构化的 Markdown 文件
- 添加双向链接
- 更新知识库索引和日志

### 第三步：文件结构生成
```
knowledge_base/
├── raw/sources/              # 原始文章
├── wiki/entities/            # 实体文件
├── wiki/concepts/            # 概念文件
├── wiki/index.md             # 知识库索引
└── wiki/log.md               # 更新日志
```

每个文件包含：元数据（标签、创建时间）、核心内容、双向链接指向相关页面。

### 第四步：Obsidian 展示知识网络
打开知识库目录，作为 Vault 拖进 Obsidian。双链网络可以自由浏览，Graph View 可以看到知识点之间的关联强度。

### 第五步：无限循环
每次导入新文档，知识网络都会自动更新。已有的页面会被补充新信息，与新内容矛盾的地方会被标注。不同文档里提到的同一个概念会被关联到同一个 Wiki 节点上。

## 安装步骤

### 第一步：安装 Obsidian

1. 打开官网 obsidian.md，下载 macOS 版本
2. 安装后打开 App，选择或创建 Vault（Vault 就是笔记仓库，本质上是一个文件夹，里面装满 Markdown 文件）

### 第二步：安装 LLM Wiki

1. 访问 https://github.com/nashsu/llm_wiki
2. 下载 macOS 的 DMG 或 App tar.gz 文件
3. 解压后把 LLM Wiki App 拖到应用程序文件夹
4. 双击打开 App，第一次使用需要新建项目
5. 点击右下角设置图标，选择模型供应商，填入 API Key
6. 支持 OpenAI、Claude、Minimax 以及任何 OpenAI 兼容的 API 接口

### 第三步：安装 Hermes Agent

**支持系统**：macOS、Linux、Windows

**安装命令**：
```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

**验证安装**：
```bash
source ~/.zshrc
hermes
```

**首次配置**：
```bash
hermes setup
hermes model  # 选择模型提供商
```

**配置知识库规则**（重要）：
```
我的知识库目录是：/Users/你的用户名/Documents/knowledge_base
规则：
1. 只有我明确说"写入知识库"时，才可以写入知识库。
2. 只有我明确说"结合知识库"时，才先检索知识库再回答。
3. 所有知识页使用 Markdown 和 [[双向链接]]。
4. 使用 LLM Wiki 文件结构管理知识库。
```

## 三条使用规则

### 规则 1：说「写入知识库」，Hermes 来整理

当有新的文档需要管理（产品文档、会议记录、学习笔记、技术方案），只需要告诉 Hermes「把这篇文章写入知识库」，它会自动完成所有整理工作。

### 规则 2：说「结合知识库」，Hermes 来检索

当需要基于资料库回答问题时，比如「根据我们已有的产品文档，介绍一下我们的技术架构」，加上「结合知识库」这几个字，Hermes 会检索知识库中的相关文档，AI 整合后给你完整回答并标注来源。

日常的普通对话（如问天气），直接回答，不会去查资料库。这样知识库不会被无关内容污染，每次检索结果也更精准。

### 规则 3：Obsidian 随时可用

wiki 目录可以直接拖进 Obsidian 当 Vault 使用。双链跳转、Graph View、全文搜索，所有 Obsidian 的功能随时可用。所有内容都是纯 Markdown，可以在任何工具之间迁移。

## 总结

- **写入知识库**：Hermes 自动整理。你说，Hermes 做。
- **结合知识库**：AI 检索、整合，回答你的问题。
- **知识库目录**：Obsidian 展示，双链自由浏览。

两个工具各有分工：
- **Hermes Agent**：自动化执行引擎，负责接收指令，提取结构化知识，按 LLM Wiki 规范创建文件
- **Obsidian**：笔记展示层，负责双向链接和知识网络可视化

用好这套工作流，你不需要手动整理笔记，不需要每次都从零开始搜索。让 AI 替你做繁琐的知识管理工作，你只需要专注于提问和探索。
