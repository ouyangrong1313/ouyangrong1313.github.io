# 从Prompt、Context到Harness，工程的三次进化与终局之战 — 拆解

**标签：** #主题/AI-Coding #主题/AI-Agent #手法/权威背书 #手法/对比冲突 #场景/公众号长文

## 核心观点

1. **Prompt / Context / Harness 是三层嵌套，不是替代关系**：Prompt 负责表达任务，Context 负责供给正确信息，Harness 负责整个系统可靠运行。
2. **Prompt 的边际收益在下降，Context 和 Harness 的重要性在上升**：模型变强后，“会不会写提示词”不再是核心差异，“能不能组织上下文和闭环验证”才是。
3. **AI 工程的关键是把“声称完成”变成“验证完成”**：lint、测试、截图、日志、观测、回写修复才是真正的质量系统。
4. **多 Agent 不是炫技，是复杂任务跨过可靠性边界后的工程解法**：当单 Agent 会迷失、自评失真、上下文耗尽时，必须靠角色拆分补稳。
5. **工程师价值正在上移**：从“写代码的人”转向“设计让 AI 稳定写代码的系统的人”。

---

## 7个分析角度

### 角度一：三次进化的主线到底是什么
- Prompt Engineering 解决“怎么说”
- Context Engineering 解决“该知道什么”
- Harness Engineering 解决“系统如何可靠运转”
- 三层组合起来，才构成可上线的 AI 工程方法

### 角度二：为什么 Prompt 开始不够用了
- 模型理解力增强，提示词技巧的边际收益下降
- 真正限制输出质量的，越来越多是缺信息而不是不会说
- Prompt 只影响单轮表达，无法天然解决多轮任务可靠性

### 角度三：Context Engineering 为什么会成为分水岭
- 模型只能看到上下文窗口内的信息
- 长上下文会发生中间遗忘，历史消息也会挤压新信息
- RAG、滚动摘要、层次记忆、单一事实来源，是实际工程中的刚需

### 角度四：Harness Engineering 为什么是生产级关键
- Prompt 和 Context 都无法防止 Agent 越权、偷懒、自评失真
- Harness 负责约束、工具、权限、验证、反馈、回滚、协作
- 没有 Harness，单次回答再好，复杂任务也会越跑越歪

### 角度五：OpenAI 与 Anthropic 的案例说明了什么
- OpenAI 的重点是上下文治理、验证闭环、技术债自动清理
- Anthropic 的重点是 Planner / Generator / Evaluator 分工
- 两个案例共同证明：系统设计比“只换更强模型”更决定成败

### 角度六：Harness 的衰变定律怎么理解
- 模型越强，很多原先必要的约束会自然变薄
- 但业务规则、权限边界、外部工具接口不会自动消失
- 好的工程师不是一味加规则，而是根据模型边界动态调厚薄

### 角度七：工程师角色为什么会被重定义
- 人的职责从写实现转向定方向、搭支架、做裁决
- 衡量标准从个人代码产出转向系统杠杆
- 谁能设计稳定高效的 Harness，谁就具备更高的放大效应

## 开头钩子（每角度3个）

### 钩子集1：框架升级型
1. 你以为 AI 工程只有提示词？真正的分水岭，早就不在 Prompt 了。
2. 从 Prompt 到 Context 再到 Harness，AI 工程其实已经悄悄经历了三次升级。
3. 如果你还在把 AI 效果不好全怪到提示词头上，问题大概率找错了。

### 钩子集2：Prompt 失效型
4. 模型越来越聪明后，提示词工程最尴尬的地方出现了：它没那么值钱了。
5. 不是你不会写 Prompt，而是 Prompt 已经不再是最关键的那一层。
6. 现在很多 AI 产出不稳定的问题，根本不是“没说清楚”，而是“系统没兜住”。

### 钩子集3：上下文治理型
7. 大模型不是不聪明，它只是像一条记忆只有几秒钟的金鱼。
8. 你给 AI 的不是知识，而是它此刻能看到的世界；这就是 Context Engineering 的本质。
9. RAG 不是高级外挂，它只是把“全部塞进去”改成了“需要时再取”。

### 钩子集4：Harness 冲击型
10. Prompt 负责把任务说清楚，Harness 负责防止 AI 把项目写坏。
11. AI 最大的问题不是不会做，而是会边做边跑偏，还坚信自己做对了。
12. 真正的生产级 AI 系统，不是更会答题，而是更会验收自己。

### 钩子集5：案例证明型
13. OpenAI 五个月做出近百万行生产代码，真正值得学的不是模型，而是那套 Harness。
14. Anthropic 发现一个扎心事实：让 Agent 自评，往往等于让 Bug 给自己打高分。
15. 当单 Agent 开始“做一半就宣布完成”，多 Agent 协作就不再是选配，而是门槛。

### 钩子集6：动态设计型
16. 模型越强，Harness 越薄，这句话会改变你设计 Agent 系统的方式。
17. 最差的工程师是给所有问题都叠规则，最强的工程师知道哪些规则迟早该删掉。
18. Harness 不是越重越好，关键在于你能不能压中模型真正的失效边界。

### 钩子集7：角色重构型
19. AI 时代工程师最值钱的能力，可能已经不是写代码，而是设计让 AI 稳定写代码。
20. “Human steer, agents execute” 不是口号，而是工程师职责重排的起点。
21. 以后拉开差距的，不是谁能写更多代码，而是谁能搭出更强的 AI 生产系统。
