## 一句话总结

**模型是商品，skill 才是资产**——AI 公司的竞争优势来自把"公司最厉害的人的判断"打包成可被 agent 复用的 skill；落地路径不是"建平台/选大模型"，而是找到具体重复工作，把顶尖人做的事写下来。

## 核心观点（5 个）

1. **公司最好的人都有"沉默专长"**：顶级销售、支持主管、财务负责人在同一份材料里看到的东西完全不同 — 这些判断力 / 品味 / 机构知识很难被捕捉和传递
2. **接数据不等于好的工作**：数据齐全后，agent 仍缺判断逻辑（"第二堵墙"）—— 它不知道你的公司**怎么做**这件事
3. **skill ≠ prompt**：prompt 是单时刻指令，skill 捕捉可重复的工作方式（含指令 / 示例 / 模板 / 检查清单 / 脚本 / 参考资料 / 经验法则）
4. **技能库 vs SOP 库的关键区别是闭环**：agent 强制使用 + 执行过程可观察 → skill 立即可见它能做什么不能什么
5. **模型越通用，分化越来自 skill**：每家用上的模型趋于相同，**真正的护城河来自你喂给它的特定知识**

## 7 个分析角度 × 3 个开头钩子

### 角度 1：现象 / 反常识

- 钩子 1：买了同样的 GPT，凭什么别家用得比你好
- 钩子 2：AI 落地的"第二堵墙" — 不是数据，是判断
- 钩子 3：为什么顶级销售能看出那通电话的走向

### 角度 2：痛点 / 矛盾

- 钩子 1：为什么写下来的 SOP 没人读
- 钩子 2：为什么"知识在老员工脑子里"是真实风险
- 钩子 3：为什么大家还在忙着接 API、买 SaaS、选大模型

### 角度 3：原因 / 为什么

- 钩子 1：为什么 skill 比 prompt 强 — skill 是"工作方式"，prompt 是"单时刻指令"
- 钩子 2：为什么"沉默专长"在 AI 时代有了显性化机会
- 钩子 3：为什么 skill 比传统 SOP 有更强的反馈循环

### 角度 4：方法 / 怎么做

- 钩子 1：Hiten 的 skill 落地三问 — 他们首先注意什么 / 什么容易被忽略 / 反复出现的问题
- 钩子 2：销售电话 / 事故复盘 / 董事会问答 — 三个具体的 skill 例子
- 钩子 3：Anthropic SKILL.md 的极简技术形态：文件夹 + 一个 SKILL.md + 几个支持文档

### 角度 5：流程 / 迭代

- 钩子 1：先从"工作本身"开始，不是"选平台"
- 钩子 2：写下来 → 用 → 看结果 → 改 → 写下来（持续循环）
- 钩子 3：从几个让重要工作更一致的 skill 开始，技能库从那里生长

### 角度 6：类比 / 跨域

- 钩子 1：skill 库 vs 教材 — 教材是给"想学"的人，skill 是给"必须用"的 agent
- 钩子 2：沉默专长 → 显性知识：经典的 Nonaka SECI 螺旋在 AI 时代重新启动
- 钩子 3：AI 时代的 onboarding 文档 = 给新人看的，skill = 给 agent 用的，本质都是"机构知识显性化"

### 角度 7：行动 / 启示

- 钩子 1：你公司的 skill 库现在有几条？如果是 0，**从一条开始**
- 钩子 2：那个经验最丰富的人，愿不愿意坐下来 — 这是 skill 项目的真正瓶颈
- 钩子 3：先动手的那批人，优势会比他们想象的大

## 我的理解

- **"模型是商品，skill 才是资产" 是 2026 年最锋利的 AI 战略判断**——和 [[从Prompt-Context到Harness-工程的三次进化与终局之战]] [[Claude-Code作者Boris-我已经不写prompt了我写loop]] 同一主线，**判断力的可传递化是 AI 时代一等价值**
- **与 [[Agent Skills 系统性综述]] [[谷歌开源 agent-skills]] 互补**：那两篇是"skill 系统的工程实现"，Hiten 这篇是"**skill 系统的战略必然性**"——为什么所有公司**必须**做这件事
- **"沉默专长"概念对 Seetong 团队很关键**：张威 / 黄松佳 / 梁添 / 李深生 / 付林青 脑子里都有大量"看不到的判断力" — 异地区域的网络处理 / 套餐失效场景 / 报警优先级，这些**写下来就是 skill，不写就是流失**
- **"先动手的那批人优势比想象的大"** —— 这个判断和 [[AI-Coding的顿悟时刻]] "工厂模式半年标配"是同一信号：**早期采纳者会拉开指数级差距**，因为 skill 库有自增强特性（用得越多越好，越好越多人用）
- **"让人坐下来写" 是真正的瓶颈** —— 跟 Seetong [[AGENTS.md]] "禁止重复造轮子 + 每次任务以 To-do List 开始" 的工程纪律是同一回事：纪律和沉淀习惯比任何工具都重要
