## 问题背景

随着 GPT、Qwen、Claude 等大模型能力持续提升，LLM-based Agent 正在从单轮问答走向更复杂的**长期任务**：多轮对话、个人助手、游戏智能体等。

在这些场景中，Agent 不仅要理解当前输入，还要持续积累过去的交互、偏好、事实变化和任务状态。

**Naive Long-Context Prompting 的问题**：
- 上下文窗口可能溢出
- Token 成本高
- 推理延迟增加
- 模型不一定能找到真正相关的证据

**Agent Memory 的核心目标**：不要让模型"每次重读全部历史"，而是让系统主动维护一套记忆机制，在需要时取回相关信息，帮助 LLM 进行更可靠的长期推理。

---

## 统一框架：四个核心组件

论文提出的统一框架将 Agent Memory 拆解为四个核心组件：

1. **Information Extraction** — 记什么？
2. **Memory Management** — 怎么维护记忆？
3. **Memory Storage** — 存在哪里、用什么结构存？
4. **Information Retrieval** — 如何取回相关记忆？

---

### 1. Information Extraction：记什么？

信息提取决定哪些内容会进入记忆系统，现有方法主要包括三类：

- **直接归档**：原始数据不做处理直接存入
- **总结式提取**：对信息进行压缩、总结后存入
- **基于图的提取**：将信息组织成图结构

---

### 2. Memory Management：怎么维护记忆？

记忆管理决定新旧记忆如何融合、演化和遗忘。论文将该过程总结为五类操作：

1. **连接相关经验**：建立记忆之间的关联
2. **整合碎片记忆**：将零散信息聚合
3. **层级间迁移**：在不同记忆层级之间移动
4. **更新已有记忆**：修改或强化旧记忆
5. **过滤无用信息**：遗忘低价值内容

---

### 3. Memory Storage：存在哪里、用什么结构存？

记忆存储可以从两个维度理解：

**组织结构**：
- **扁平式存储**（JSON、队列）
- **层级式存储**（长短期、树结构）

**表示方式**：
- **基于向量的存储**
- **基于图的存储**

---

### 4. Information Retrieval：如何取回相关记忆？

信息检索决定当前 query 到来时，系统如何从记忆中找到最有用的信息。分为四类：

| 检索方式 | 说明 |
|---------|------|
| **词汇匹配检索** | BM25、Jaccard，精确匹配实体、名称和关键词 |
| **向量检索** | 向量余弦相似度、ANN 算法 |
| **结构检索** | 利用图或树中的显式连接，通过邻居扩展、图遍历找到相关信息 |
| **LLM辅助检索** | 让 LLM 参与检索过程，识别关键信息或直接判断记忆相关性 |

## 实验设计

### 数据集

- **LOCOMO**：人类长期对话记忆数据集，问题覆盖单跳、多跳、时间推理和开放域知识等类型
- **LONGMEMEVAL**：用户与 AI 长期交互记忆数据集，用于评估信息提取、多会话推理、知识更新和时间推理等能力

### 评测方法

统一复现并比较 **10 个代表性 Agent Memory 方法**，实验包括：
- 整体性能对比
- Token 消耗情况及性能-成本权衡
- 上下文扩展性分析
- 证据位置敏感性分析
- 不同底层 LLM 下的表现对比

---

## 主要结果与发现

### 发现1：层次化方法表现突出

MemTree、MemoryOS、MemOS 等层次化或树状方法表现突出，说明**多层结构能够同时保留高层摘要和底层证据**，更适合复杂长期任务。

### 发现2：粗粒度处理降本提效

将多轮对话作为一个整体进行处理可以显著降低 token 消耗，适当的粗粒度处理反而可能提升记忆效果。

### 发现3：上下文扩展性有限

当上下文规模扩展到 200% 时，几乎所有方法都会出现性能下降。相比之下，采用更明确层次管理的方法通常更稳定。

### 发现4：证据位置敏感

多数方法存在证据位置敏感性：当关键证据位于更早会话时，很多方法更容易被后续信息干扰而检索失败。

### 发现5：底层 LLM 仍关键

现有记忆架构仍然依赖底层 LLM 的推理能力。从 Qwen2.5-7B 扩展到 72B 后，多数方法都有明显提升。

## 新 SOTA 算法

基于上述发现，组合了：
- **MemTree/MemOS** 的树状组织能力
- **MemoryOS** 的分层存储架构

设计出一个新的低 token 开销 Agent Memory 框架。
