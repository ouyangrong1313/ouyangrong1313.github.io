# 聪明人用AI做的，是给自己出题 - 拆解

## 核心观点

1. **延迟命名** — 标签是思考的终点，逼迫大脑深度理解后再给概念命名
2. **故事 > 定义** — 理解一件事的标志是能否让它变回故事，而非复述定义
3. **定义是压缩，故事是解压缩** — 定义丢弃了维度、情绪和后果
4. **反定义思维训练** — 平庸问题只能得到平庸答案，问题必须足够具体、足够偏离轴心
5. **AI训练大脑** — 用AI设计思考难度，而非直接给答案

## 三个精妙设计

| 设计 | 含义 |
|------|------|
| 刻意设置难度 | 太通俗没启发，太艰深过于遥远 |
| 延迟命名 | 逼迫大脑进入"深度理解" |
| 反向费曼 | 让你听不懂名字，却懂了逻辑本身 |

## Amanda Askell 提示词模板

**三步法**：
1. 找一个"本科生不了解、研究生才懂"的小众概念
2. 用三段式故事阐释，故事结束前绝不提概念名
3. 最后揭晓名称，解释故事如何对应概念的每个要素

## 核心 Prompt

```
我想搞懂[X]，但你别直接告诉我定义。我们这样来：
1、先找一个跟[X]很相近、但更冷门的原理；
2、用三段话编一个故事，把这个原理演出来——故事里不许出现概念的名字，要有冲突，要有压力，让我感受到当事人在做什么样的取舍；
3、故事讲完先别揭晓。先问我："你觉得这个故事在讲什么道理？" 等我答完，再告诉我[X]叫什么；
4、最后回头对照——故事里哪个情节对应了概念的哪一个核心要素，一条条说给我听。
```

## 7个分析角度

1. **学习方法** — 从定义驱动到问题驱动
2. **AI使用技巧** — 用AI出题而非用AI答题
3. **认知科学** — 压缩vs解压缩、触感vs定义
4. **思维训练** — 反定义的必要性
5. **费曼学习法** — 正向vs反向
6. **提问艺术** — 如何问出好问题
7. **大脑作为湿件** — 影像思考与语言描述

## 开头钩子库（14个）

### 角度1：学习方法
1. "聪明人用AI做的，是给自己出题"
2. "理解一件事的标志，不再是你能不能复述它的定义，而是你能不能让它变回一个故事"
3. "定义是压缩，故事是解压缩"

### 角度2：AI使用技巧
4. "下次想真正吃透一个概念，别再问AI'这是什么'"
5. "让AI给你出一道题，而非给你一个答案"
6. "用AI训练自己的大模型——你的大脑"

### 角度3：认知科学
7. "标签是思考的终点，延迟命名是逼迫大脑深度理解"
8. "你不是在'懂了'贝叶斯更新，你是在那个不断修正判断的侦探身上，经历了一次贝叶斯更新"
9. "大脑是一个湿件，而不是硬件"

### 角度4：思维训练
10. "平庸的问题只能得到平庸的答案"
11. "你的问题必须足够具体、足够偏离轴心"
12. "最高效的思维训练，往往是反定义的"

### 角度5：费曼学习法
13. "费曼法是'讲给小孩听'，反向费曼是'让AI把概念讲到你听不懂名字，却懂了逻辑本身'"
14. "此为触感，而非定义"
