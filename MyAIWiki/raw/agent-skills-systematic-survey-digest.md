# Agent Skills 系统性综述 — 拆解笔记

---

## 核心观点（5个）

1. **技能是 Agent 的肌肉记忆**：不是模型不够聪明，而是缺乏把重复经验变成可复用技能的能力
2. **技能 = 三元组 S=(M,R,C)**：主指令文档 + 辅助资源 + 触发条件
3. **技能获取有四条路径**：人类专家手写、从经验提炼、即时构建、外部挖掘，四条路组合使用最强
4. **检索召回率 ≠ 执行成功率**：语义近的技能不一定能跑起来，这是新瓶颈
5. **技能生命周期管理比技能本身更重要**：获取只是起点，需要持续修订、验证、进化、治理

---

## 七个分析角度

### 1. 定义与本质
- 技能三元组：S = (M, R, C)
- M=主指令文档（做什么），R=辅助资源（用什么），C=触发条件（何时用）
- 三种类型：纯文本型、纯代码型、混合型（Claude Code的CLAUDE.md就是混合型）

### 2. 获取路径（四条路）
- **人类专家手写**：精度最高但扩展性差，作为种子层
- **从经验提炼**：主流路径，从成功轨迹中提取（Voyager/Reflexion/ExpeL）
- **即时构建**：面对新任务让LLM直接生成（CREATOR/ToolMakers）
- **外部资料挖掘**：从文档/代码库/Kaggle提取，冷启动首选

### 3. 检索策略（四类）
- 语义向量检索：最常用但语义近≠适用
- 关键词检索：精确但简单，做补充过滤
- 生成式检索：融入推理但覆盖率难保证
- 结构化检索：利用层级/依赖关系，适合大规模库

### 4. 进化机制（五个环节）
- 修订（Revision）：修改技能本身内容而非记录错误
- 验证（Validation）：通过测试才进正式库
- 策略耦合（Policy Coupling）：技能库成为可训练参数
- 仓库级进化：单个技能→整个仓库治理
- 运行时治理：警惕"投毒技能"风险

### 5. 技能分类
- 纯文本型：可读性强，执行确定性弱
- 纯代码型：执行可靠，维护成本高
- 混合型：兼顾二者，一致性维护最复杂

### 6. 生态现状
- SkillsMP：70万+ 技能
- SkillNet：30万+ 技能
- ClawHub：4万+ 技能
- 技能正在成为独立基础设施层

### 7. 产品启示
- 下一个竞争力 = 技能管理能力，而非模型本身
- 需要：技能存储 + 检索系统 + 测试框架 + 版本管理 + 安全审核
- 技能生命周期：表示→获取→检索→进化（完整闭环）

## 开头钩子（21个）

### 角度1：定义与本质（3个）
1. 你让AI写代码，它第二天又从头推理——问题不在模型，在它没有"肌肉记忆"
2. Agent缺的不是智力，是把经验变成技能的能力——这就是为什么它每次都从零开始
3. 技能三元组S=(M,R,C)：主指令+辅助资源+触发条件，定义比你想的要精确

### 角度2：获取路径（3个）
4. 从成功轨迹中提取可复用模式——Voyager在Minecraft里就是这么做的
5. 四条获取路径：手写、提炼、即时构建、外部挖掘——最强技能库是组合的结果
6. 冷启动不用等积累：从外部资料挖掘，Kaggle方案、API文档都能变成技能

### 角度3：检索策略（3个）
7. 70万技能在手，核心问题变了——不是有没有，而是能不能在正确时刻找到并激活
8. 检索召回率不等于执行成功率：语义近≠能跑起来，这条鸿沟才是真正瓶颈
9. 四种检索策略：语义向量、关键词、生成式、结构化——没有最优，只有适用

### 角度4：进化机制（3个）
10. 学会开车是一回事，越开越好是另一回事——技能获取和进化是两件事
11. 修订不是记录错误，是修改技能本身——Memento-Skills就是这么做的
12. "投毒技能"风险：第三方技能文档可能隐藏恶意逻辑，被Agent当作可信操作执行

### 角度5：技能分类（3个）
13. 纯文本型：可读性强但执行确定性弱；纯代码型：执行可靠但维护成本高
14. Claude Code的CLAUDE.md+辅助脚本，本质上是混合型技能系统
15. 混合型兼顾可读性和可执行性，但一致性维护最复杂——这是核心挑战

### 角度6：生态现状（3个）
16. SkillNet 30万+、ClawHub 4万+、SkillsMP 70万+——技能已成独立基础设施层
17. SkillsMP已有70万+技能——规模带来的问题不再是数量，而是检索和选择
18. 技能平台崛起意味着：技能不再是Agent产品的附属，而是独立赛道

### 角度7：产品启示（3个）
19. Agent的下一个竞争力 = 技能管理能力，而非模型本身——模型是大脑，技能是肌肉记忆
20. 你需要的不只是技能存储，还需要检索系统、测试框架、版本管理、安全审核
21. 127篇文献梳理的完整框架：表示→获取→检索→进化，缺任何一环都不完整
