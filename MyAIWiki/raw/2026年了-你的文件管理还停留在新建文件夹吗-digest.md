# 2026年了，你的文件管理还停留在新建文件夹吗 — 拆解

## 核心观点

1. **文件管理本质是信息管理** — 万物皆比特，具体写了什么不重要，传递了什么信息才是最重要的
2. **三类信息** — 工具类（文件）、任务类（文件）、观点知识类（文件）
3. **Inbox 入口统一** — 减少 Inbox 入口缓解焦虑，防止信息过载
4. **统一命名是基础** — 没有合理的命名，就是没有意义的文件
5. **文件夹永不过时** — 杜威十进制图书管理法 + PARA System
6. **输出倒逼输入** — 复用复用再复用，最终内化成本能
7. **321 备份原则** — 3份副本、2种介质、1份异地

## 7 个分析角度

### 角度1：本质论 — 文件管理是信息管理
约翰·惠勒的 "It From Bit" 观点：万物皆比特，一切交互本质都是信息交互。文件只是信息的载体，核心是传递信息。

### 角度2：分类论 — 三类信息
- 工具类信息：高复用图表、音视频、文字素材
- 任务类信息：大大小小需要跟进的任務
- 观点知识类信息：文娱影视、课程、文章等

### 角度3：入口论 — Inbox 统一
减少 Inbox 入口确保信息来源唯一性。用 Eagle 管理文件类 Inbox，Notion 管理知识类 Inbox。

### 角度4：命名论 — 文件名是基础名片
命名考虑：概括性、可读性、适用性、检索便利性、软件兼容性。维度：领域_项目_版本_执行人_标签_备注_地点_时间。

### 角度5：体系论 — 两大文件夹管理法
- 杜威十进制：001-999 编号体系
- PARA System：Project/Area/Resource/Archive

### 角度6：复用论 — 输出倒逼输入
文件是拿来用的，不是存起来供起来展示的。反复复用，内化成本能，张口即来。

### 角度7：备份论 — 321 原则
3份副本、2种存储介质、1份异地存储。合理冗余备份关键时刻能救命。
