> 来源：陈进 / 公众号 腾讯云开发者（2026-06）；二手解读 Agent Loop 社群手册 + 自述 30 天 SELF Protocol 实验

## 一句话总结
> **Agent Loop 是"让 AI 自己转起来"的新范式，但 8 个真实问题没解（软目标停止、同模型 Maker-Checker 共病、护栏分层、记忆失焦、理解力腐蚀、拓扑选型、胡说防不住、多 Agent 成本）；SELF Protocol 是一种"白盒治理薄壳"的草稿式回应。**

## 核心观点 8 条
1. **范式迁移**：Prompt → Harness → Loop，人从"喂一句话"升级到"设计循环 + 护栏"
2. **Stopping Condition 第一**：写循环前先写"什么算干完了"，模糊目标让 AI 自欺
3. **Context 自动拼**：不是写出来的，是从失败日志/文件树/调用记录动态组装的
4. **失败是输入不是终点**：把错误堆栈喂回下一轮，把"出错就停"翻成"出错就学习"
5. **6 种多 Agent 拓扑**：顺序流水线 / 协调者-工作者 / 扇出合并 / 生成-验证 / 共享状态 / 辩论对抗
6. **SELF 是治理薄壳**：不抢 Loop 框架位置，专治"转起来怎么不跑偏"
7. **诚实 disclaimer 比假完成重要**：软目标场景强制标"计划级 / 我没法验证"
8. **8 个真痛点都没标准答案**：作者 30 天单 Agent 体感，无对照组，求评论区拍砖

## SELF vs Loop 5 件套（治理相关）

| 件套 | Loop 自带 | SELF 补强 | 评估 |
|---|---|---|---|
| Memory | Markdown / DB 突破上下文 | 三层分级 l0/l1/l2 + 答前强制翻笔记 | 30 天有效，仍有失焦 |
| Sub-agents | Maker-Checker | 单 Agent 内置 pre-publish review 角色卡 | 强于无验收，但同模型盲区共存 |
| Guardrails | 资源类（预算/迭代/无进展） | + 认知类（不胡说/不失忆/不污染） | 拦下 1 次编造，触发率仍不稳 |
| Skills | 可复用指令 | + "失败转技能"自动沉淀 | 30 天攒 9 条，但同类失败仍反复 |
| Stopping Condition | 硬目标（写测试通过） | 软目标 + 诚实分级 disclaimer | 治不了软目标，但不会假装完成 |

## 8 个真痛点（速查）

| # | 痛点 | 作者土办法 | 待解 |
|---|---|---|---|
| 1 | 软目标停止条件 | LLM judge + 诚实 disclaimer | judge 漂移（上午 0.85 / 下午 0.6） |
| 2 | Maker-Checker 同病 | 仍用同模型 | 是不是该用不同模型？规则引擎？ |
| 3 | 护栏分层 | 认知类可插拔 / 资源类写死 | 对不对？ |
| 4 | 记忆给多大 | 三层 l0/l1/l2 按需展开 | 仍有失焦，token 截断 vs 相关性打分 vs 时间衰减？ |
| 5 | 理解力腐蚀 | 暂无 | 怎么对冲？ |
| 6 | 拓扑选型 | 试过"辩论对抗"反而更糊涂 | 哪些拓扑看着美用着崩？ |
| 7 | AI 一本正经胡说 | pre-publish review 清单 | 拦下 1 次，触发率不稳 |
| 8 | 多 Agent 成本 | 便宜模型路由 + 中间产物不进 Prompt + 限轮数 | 模型分级？压缩？缓存？ |

## SELF 关键事实卡
- ~1500 行 Python + Markdown 约定
- Knot Skill 33837 · v1.6.7
- 30+ 天单 Agent 单样本无对照
- 268 次 LLM 调用 / 20 失败案例 / 314 学习记录
- 拦下编造数据 1 次（自动周报循环里）
- 4 档入口：只读 → 一次性 → 候选写入需拍板 → 长期托管
- 输入"关闭 SELF"即停，零变更

## 核心金句
- "别再死磕 Prompt 怎么写了，去设计一个能让 AI 自己转起来的循环（Loop）。"
- "模糊的目标会让 AI 自欺欺人，可验证的目标才是真护栏。"
- "失败是输入，不是终点。"
- "Agent Loop 教我怎么让 AI 自己转起来，SELF 是我回答转起来怎么不跑偏的那一层薄壳——前者是骨架，后者是润滑层。"
- "我特别需要被指出'这里你想偏了'的时刻。"
