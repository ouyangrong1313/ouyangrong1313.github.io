# Bug Fix Prompt Template

## 使用方法
```
把这段粘贴给 Claude Code，配合你的 bug 描述
```

## 模板

```
我需要修复一个 bug，请按照以下步骤进行：

## 1. 复现问题
- 描述：{bug 描述}
- 复现步骤：{步骤1、步骤2、步骤3}
- 预期行为：{应该是什么}
- 实际行为：{实际是什么}

## 2. 分析根因
请先：
1. 读取相关代码文件
2. 追踪数据流和调用链
3. 找出根本原因（不是表象）

## 3. 修复方案
提出修复方案，包含：
- 修改的文件
- 具体的代码变更
- 为什么这样修

## 4. 验证
- 如何验证 bug 已修复
- 需要测试的场景

---
输出格式：
- 根因分析
- 修复代码（用 ``` 包裹）
- 验证步骤
```

## 示例

```
我需要修复一个 bug，请按照以下步骤进行：

## 1. 复现问题
- 描述：用户登录后立即退出
- 复现步骤：1. 打开APP 2. 输入账号密码 3. 点击登录 4. 跳转后立即跳回登录页
- 预期行为：登录成功后停留在首页
- 实际行为：登录后马上退出到登录页

## 2. 分析根因
请先：
1. 读取 LoginVC.m 和 STUserManager.m
2. 检查 token 存储和验证逻辑
3. 找出为什么 token 没有正确保存或验证

## 3. 修复方案
提出修复方案，包含：
- 修改的文件
- 具体的代码变更
- 为什么这样修

## 4. 验证
- 如何验证 bug 已修复
- 需要测试的场景
```
